# 项目级 AI 协作规则

> 三段式结构：硬规则（必须遵守）→ 项目事实（参考用）→ 用户画像（参考用）

## 一、硬规则（AI 必须遵守）

### 1. 工作模式
- 先计划 → 再执行 → 后审查
- 大改动前先出计划，你确认后再执行
- 执行完成后必须审查，逐项核对计划

### 2. 改代码前先查文档
- **任何代码改动前，必须先查阅 `docs/guide/` 中相关文档**
- 不确定改什么时，先读 `docs/guide/data-flow.md` 确认链路
- 不能仅凭记忆或之前的上下文判断

### 3. Git 约定
- 不擅自提交、不擅自推送
- 提交前先解释改了什么、哪些文件会提交、提交信息为什么这样写
- 优先 `git add <具体文件>`，避免 `git add .`
- 提交后运行 `git status` 验证状态

### 4. 文档协作
- 不擅自创建或修改文档，先问你
- 涉及新增 API 路由 → 检查 `main.py` 是否注册 router
- 涉及新增字段 → 检查前后端类型是否同步
- 涉及数据库索引或唯一约束迁移 → 检查具体字段列表

### 5. 质量门禁
- 涉及删除/替换公共模块 → 必须做引用扫描，列出所有引用点
- 执行完成后 → 启动服务并调用至少一个涉及修改的接口验证
- 不能只做语法检查（py_compile / tsc），必须验证运行时行为

### 6. 报错处理
- 出现报错时，先解释报错含义，再给解决步骤

## 二、项目事实（参考用）

### 技术栈
- 后端：Python FastAPI + SQLite
- 前端：React Native + Expo + TypeScript
- 行情源：efinance（主源）+ 新浪（fallback）
- AI：火山方舟 DeepSeek
- 导航：React Navigation 6（Tab + Stack）
- 状态：Zustand（store）+ AuthContext

### 产品定位
AI Stock App 是面向个人投资者的 AI 股票分析助手，不是投资交易软件，也不是完整投研工作台。
核心能力：自选股票、AI 问股、分析记录、个人账号。

### 目录说明
- `backend/` — Python FastAPI 后端
- `mobile-app/` — React Native 移动端
- `stock-analysis-reference/` — 只读参考项目，不修改、不提交
- `docs/guide/` — 常用参考文档（数据流、项目结构、架构总览）
- `docs/archive/` — 历史文档归档（计划、总结、学习笔记）

### 文档索引
| 文档 | 用途 |
|------|------|

## 三、用户画像（参考用）

### 学习目标
- **目标岗位：** AI 应用开发工程师
- **核心能力：** 用 AI 编程工具构建产品，理解 AI 应用架构（Prompt 工程、RAG、Agent、流式响应）
- **前端/后端知识：** 辅助理解，不是主攻方向
- **工程化实践：** 理解概念和为什么重要，具体实现由 AI 协助完成

### 已掌握的基础
- Python FastAPI 后端开发（路由、鉴权、数据库、服务层）
- React Native 移动端开发（导航、状态管理、组件抽离、自定义 Hook）
- SQLite 数据库设计与迁移
- Git 版本管理与迭代规划
- AI 协作开发（计划 → 执行 → 审查）

### 学习方式
- **每做一步，同步讲解背后的工程原理和生产级实践**
- **先理解再动手，重视"为什么"**
- 新概念优先联系已掌握的知识，用工程类比说明
- 鼓励用自己的话复述理解，并及时纠正
- 刚学完的内容不重复复习，进入新阶段或间隔较久后主动安排简短复习
- 可以适当插入实用技巧、工程经验和进阶知识
- 工具学习（MCP、Skill、Agent 工作流）作为副线，不中断项目主线

### 沟通偏好
- 使用中文交流
- 解释清晰直接，不需要从零开始的"新手解释"
- 多互动，主动询问哪里不懂
- 聊天和学习时自然对话，不必过度严肃
- 涉及代码修改、文档写入、提交、推送时必须先确认
