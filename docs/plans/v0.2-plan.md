# v0.2 基础分析增强版计划

## 1. 版本定位

v0.2 的目标是让 AI Stock App 从“能跑通分析闭环”升级为“能提供更有依据的基础股票分析”。

核心原则：

- 借鉴原参考项目思想，但不照搬代码。
- 先做最小可用增强，不一次性追求完整专业量化系统。
- 每个阶段都要能独立验收。

## 2. 功能范围

### 必做范围

1. 技术指标基础版
   - 获取最近至少 20 个交易日历史行情。
   - 计算 MA5 / MA10 / MA20。
   - 计算当前价相对 MA5 / MA10 / MA20 的乖离率。
   - 将指标写入报告 `indicators_json`。
   - 移动端报告详情页展示指标。

2. 动态趋势判断
   - `MA5 > MA10 > MA20`：多头排列。
   - `MA5 < MA10 < MA20`：空头排列。
   - 其他：震荡 / 趋势不明。

3. 动态评分
   - 基础分 50。
   - 多头排列加分，空头排列减分。
   - 根据涨跌幅和乖离率调整分数。
   - 最终分数限制在 0-100。

4. 动态操作建议
   - `score >= 75`：关注。
   - `50 <= score < 75`：观望。
   - `score < 50`：谨慎。
   - 暂不使用“买入 / 卖出”等强交易词。

5. 报告内容增强
   - 增强 `summary`。
   - 动态生成 `risks_json`。
   - `indicators_json` 增加 MA、乖离率、均线排列、评分原因等字段。
   - 移动端展示当前价格、涨跌幅、均线、乖离率、评分原因和风险提示。

### 选做范围

1. fallback 最小版
   - 保留 `efinance` 为主数据源。
   - 历史行情获取失败时返回明确错误。
   - 预留 fallback 结构，但暂不做完整 `DataFetcherManager`。

2. 问股最小版
   - 输入股票代码。
   - 后端返回基于行情和指标的简短分析。
   - 暂不做多轮会话、流式输出、大模型调用和聊天历史。

3. 我的页基础整理
   - 新增或改造 `MineScreen`。
   - 保留后端地址设置。
   - 增加版本信息和登录注册占位。
   - 不做真实账号体系。

## 3. 不做范围

v0.2 暂不做：

- 完整登录注册。
- JWT / token 鉴权。
- 自选股按用户隔离。
- 报告按用户隔离。
- 完整 AI Agent。
- 流式聊天。
- MACD / RSI 完整接入。
- 港股 / 美股等多市场支持。
- 完整 DataFetcherManager。
- Jinja2 报告模板系统。
- 推送提醒。

## 5. 数据库影响

v0.2 先不改数据库表结构。

新增指标写入已有字段：

```text
reports.indicators_json
```

原因：

- 不需要数据库迁移。
- 兼容旧报告。
- 前端按可选字段展示即可。

注意：

- 旧报告没有 MA 字段，前端必须允许缺失。

## 7. 学习重点

v0.2 主要学习：

1. 服务层拆分：API 不直接堆业务逻辑。
2. 技术指标计算：MA 和乖离率。
3. 报告生成规则：score / action / summary。
4. JSON 字段扩展：不改表结构扩展报告。
5. 兼容旧数据：旧报告没有新字段也不能崩。
6. 最小 fallback 思维：主路径失败时如何降级。
7. 问股最小闭环：从占位页变成可用功能。

## 8. 风险点

1. 历史行情接口不稳定
   - `efinance` 历史行情字段可能变化，或部分股票取不到数据。
   - 处理方式：先做字段检查和清晰错误，不强行兜底。

2. 旧报告缺少新字段
   - 旧报告 `indicators_json` 没有 MA 字段。
   - 处理方式：前端所有新增字段按可选值展示。

3. v0.2 范围变大
   - 同时做指标、fallback、问股、我的页，容易过大。
   - 处理方式：按阶段提交，每阶段独立验收。

4. 操作建议用词风险
   - “买入 / 卖出”容易被理解为投资建议。
   - 处理方式：使用“关注 / 观望 / 谨慎”等弱建议词。
