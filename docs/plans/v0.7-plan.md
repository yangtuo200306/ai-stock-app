# v0.7 功能定型与交互补齐计划

## 1. 版本目标

v0.7 的目标是在 UI/UX 大优化前完成产品功能定型，补齐 v0.6 后暴露的操作断点。

本版本重点不是大幅美化页面，而是让以下四个底部 Tab 的职责稳定：

```text
自选 | 问股 | 记录 | 我的
```

完成后，AI Stock App 应具备更完整的业务闭环：

1. 未登录用户能明确知道需要登录。
2. 问股页可以主动新建会话。
3. 记录详情可以回到问股页继续追问。
4. 多轮追问后记录列表按最近更新时间排序。
5. 自选页能看到最近一次分析摘要。
6. 报告作为记录详情的一种类型固定下来。
7. API 错误结构和前端错误展示初步统一。
8. 清理明显未使用页面和重复逻辑。
9. `ask.py` 职责降低，为后续维护做准备。

## 2. 本版本范围

### 2.1 纳入范围

1. 未登录态业务页面统一引导登录。
2. 问股页增加“新建会话 / 重新开始”按钮。
3. 记录详情支持跳转问股页继续追问，并携带 `session_id`。
4. records 增加 `updated_at`，追问后按最近更新时间排序。
5. 自选页展示最近一次分析摘要。
6. 报告固定为记录详情中的一种类型。
7. 初步统一 API 错误结构和前端错误展示。
8. 清理未挂载页面和重复逻辑。
9. 抽出 ask service，降低 `ask.py` 职责复杂度。
10. 确认底部导航和页面职责。

### 2.2 不纳入范围

1. 不做 UI/UX 大美化。
2. 不做完整历史会话列表。
3. 不做问股页自动恢复所有历史会话。
4. 不做记录详情页内直接输入追问。
5. 不做复杂 API 错误中间件。
6. 不做全量 OpenAPI 类型生成。
7. 不做数据库旧 `analysis` 记录迁移。
8. 不做股票搜索、拼音识别或全市场简称识别。
9. 不做流式输出。
10. 不 push v0.6 / v0.7，等待用户确认。

## 3. 后端计划

### 3.1 records 增加 updated_at

涉及文件：

- `backend/app/database.py`
- `backend/app/api/ask.py`
- `backend/app/api/records.py`

计划：

1. 在 `records` 表增加 `updated_at` 字段。
2. 迁移采用两步方式：
   - `ensure_column(connection, "records", "updated_at", "TEXT")`
   - 将已有空值更新为 `created_at`。
3. 新增记录时让 `updated_at` 有值。
4. 追问更新同一条问股记录时，同步更新 `updated_at = CURRENT_TIMESTAMP`。
5. `GET /api/records` 改为按 `updated_at DESC, id DESC` 排序。
6. `GET /api/records` 返回 `updated_at`。
7. `GET /api/records/{record_id}` 返回 `updated_at`。

### 3.2 自选页最近一次分析摘要

涉及文件：

- `backend/app/api/stocks.py`

计划：

1. `GET /api/stocks` 返回每只自选股的最近一条相关记录。
2. 查询来源为 `records` 表。
3. 匹配条件：
   - 当前用户；
   - `records.stock_code = stocks.code`；
   - `record_type IN ('ask', 'analysis', 'report')`。
4. 最近记录排序：`updated_at DESC, id DESC`。
5. 每只股票返回新增字段：
   - `latest_record_id`
   - `latest_record_type`
   - `latest_summary`
   - `latest_updated_at`
6. 无记录时以上字段返回 `None` 或缺省值。

### 3.3 报告作为记录详情的一种类型

涉及文件：

- `backend/app/api/analysis.py`

计划：

1. 自选分析成功生成报告后，新增记录类型调整为 `report`。
2. 新记录字段：
   - `record_type = "report"`
   - `title = "报告：股票名（代码）"`
   - `summary = report["summary"]`
   - `report_id = 当前报告 ID`
   - `metadata_json` 继续保存价格、涨跌幅、评分、建议、趋势、`task_id`。
3. 旧 `analysis` 记录继续兼容，不做数据迁移。

### 3.4 初步统一 API 错误结构

涉及文件：

- `backend/app/database.py`
- `backend/app/api/auth.py`
- `backend/app/api/ask.py`
- `backend/app/api/market.py`
- `backend/app/api/analysis.py`
- `backend/app/api/records.py`
- `backend/app/api/reports.py`

后端错误结构约定：

```json
{
  "error_code": "ERROR_CODE",
  "message": "用户可读错误信息"
}
```

计划重点覆盖：

1. 未登录 / token 无效。
2. 登录注册失败。
3. 问股输入错误。
4. 会话不存在。
5. 同一会话切换股票。
6. 行情失败。
7. record 不存在。
8. report 不存在。
9. task 不存在。

说明：

1. 本版本不做全局异常中间件。
2. 本版本不统一所有成功响应结构。
3. 本版本不设计复杂错误码枚举文件。

### 3.5 抽出 ask service

涉及文件：

- `backend/app/api/ask.py`
- `backend/app/services/ask_service.py`

计划：

1. 新增 `backend/app/services/ask_service.py`。
2. 从 `ask.py` 移出以下辅助函数：
   - 规则回答构建；
   - 会话标题构建；
   - 创建会话；
   - 查询会话；
   - 更新会话；
   - 写入消息；
   - 查询消息；
   - 写入或更新问股记录。
3. `ask.py` 保留：
   - `APIRouter`；
   - 请求模型；
   - 响应模型；
   - `/api/ask` 主流程。
4. 不改变 `/api/ask` 路径和响应字段。
5. 新增 service 文件后不需要在 `main.py` 注册 router。

## 4. 前端计划

### 4.1 类型更新

涉及文件：

- `mobile-app/src/types/index.ts`

计划：

1. `Stock` 增加最近记录摘要字段：
   - `latest_record_id?: number | null`
   - `latest_record_type?: string | null`
   - `latest_summary?: string | null`
   - `latest_updated_at?: string | null`
2. `RecordItem` 增加 `updated_at?: string`。
3. `RecordDetail` 增加 `updated_at?: string`。
4. `RootTabParamList` 的 `问股` 参数增加：
   - `sessionId?: string`
   - `initialQuestion?: string`
   - `initialMessages?: AskMessage[]`

### 4.2 问股页新建会话

涉及文件：

- `mobile-app/src/screens/AskScreen.tsx`

计划：

1. 增加“新建会话”按钮。
2. 按钮在已有会话或已有消息时显示。
3. 点击后清空：
   - `sessionId`
   - `messages`
   - `latestResult`
   - `question`
   - `error`
4. 不调用后端接口。
5. 原发送逻辑保持：
   - 有 `sessionId`：继续追问；
   - 无 `sessionId`：创建新会话。

### 4.3 问股页接收记录详情传入的 session_id

涉及文件：

- `mobile-app/src/screens/AskScreen.tsx`
- `mobile-app/src/types/index.ts`

计划：

1. 问股页读取路由参数 `sessionId`。
2. 如果传入 `sessionId`，设置为当前会话。
3. 如果传入 `initialMessages`，恢复到 `messages`。
4. 如果没有传入消息，至少保留 `sessionId`，下一次发送作为追问。
5. 不新增后端接口加载历史消息。

### 4.4 记录详情继续追问

涉及文件：

- `mobile-app/src/screens/RecordDetailScreen.tsx`

计划：

1. 当记录满足以下条件时显示“继续追问”按钮：
   - `record_type === 'ask'`
   - 存在 `session_id`
2. 点击后跳转到底部 Tab `问股`。
3. 跳转参数携带：
   - `sessionId = record.session_id`
   - `initialMessages = record.messages ?? []`
4. 不在记录详情页直接增加输入框。

### 4.5 未登录态统一引导登录

涉及文件：

- `mobile-app/src/contexts/AuthContext.tsx`
- `mobile-app/src/screens/WatchlistScreen.tsx`
- `mobile-app/src/screens/AskScreen.tsx`
- `mobile-app/src/screens/RecordListScreen.tsx`
- `mobile-app/src/screens/RecordDetailScreen.tsx`
- `mobile-app/src/screens/TaskStatusScreen.tsx`
- `mobile-app/src/screens/ReportDetailScreen.tsx`

计划：

1. 业务页面读取 `useAuth()`。
2. 如果 `isLoading`，显示加载态。
3. 如果未登录，显示统一提示：
   - 标题：`请先登录`
   - 描述：`登录后可以使用自选、问股和记录功能。`
   - 按钮：`去登录`
4. 点击 `去登录` 跳转到 `我的` Tab 的 `Login` 页面。
5. 最小实现中允许各页面局部实现相同 UI，不先抽组件。
6. 我的页和登录页不做拦截。

### 4.6 自选页展示最近一次分析摘要

涉及文件：

- `mobile-app/src/screens/WatchlistScreen.tsx`

计划：

1. 读取 `GET /api/stocks` 新增字段。
2. 自选卡片展示最近记录摘要。
3. 展示内容：
   - 最近记录类型；
   - 摘要，最多两行；
   - 最近更新时间。
4. 没有记录时显示：`暂无分析记录`。
5. 本版本不增加点击摘要进入记录详情，避免跨 Stack 导航复杂化。

### 4.7 记录列表和详情支持 report 类型

涉及文件：

- `mobile-app/src/screens/RecordListScreen.tsx`
- `mobile-app/src/screens/RecordDetailScreen.tsx`

计划：

1. 记录列表类型标签增加：
   - `ask` -> `问股`
   - `report` -> `报告`
   - `analysis` -> `分析`，用于兼容旧数据。
2. `report` 类型使用单独颜色。
3. 记录详情中：
   - `report` 显示为报告记录；
   - 展示摘要；
   - 有 `report_id` 时继续显示 `查看完整报告` 按钮。
4. 旧 `analysis` 记录保持可读。

### 4.8 API client 统一错误展示

涉及文件：

- `mobile-app/src/api/client.ts`
- 相关业务页面 catch 逻辑

计划：

1. `apiGet` / `apiPost` / `apiDelete` 判断 `response.ok`。
2. 非 2xx 响应时解析错误消息：
   - 优先 `detail.message`
   - 其次 `message`
   - 其次字符串 `detail`
   - 最后默认错误文案
3. 抛出 `Error(message)`。
4. 页面 `catch` 中展示 `error.message`。
5. 401 不在 API client 内自动跳转，由页面登录引导处理。

### 4.9 清理重复页面和重复逻辑

涉及文件：

- `mobile-app/src/screens/SettingsScreen.tsx`
- `mobile-app/src/screens/MarketScreen.tsx`
- `mobile-app/src/screens/MineScreen.tsx`
- `mobile-app/src/screens/WatchlistScreen.tsx`
- `mobile-app/src/screens/AskScreen.tsx`

计划：

1. 搜索确认 `SettingsScreen.tsx` 未被引用后删除。
2. 搜索确认 `MarketScreen.tsx` 未被引用后删除。
3. `MineScreen.tsx` 使用 `STORAGE_KEYS.BACKEND_URL`，不再单独定义 `BACKEND_URL_STORAGE_KEY`。
4. `WatchlistScreen.tsx` 合并重复的 `useNavigation` 导入。
5. 修复 `AskScreen` 加入自选失败仍提示成功的问题：
   - 成功或重复加入：提示已加入自选；
   - 真实失败：提示加入失败。

## 5. 导航职责确认

底部 Tab 保持：

```text
自选 | 问股 | 记录 | 我的
```

职责如下：

1. 自选：自选管理、发起分析、跳转问股、展示最近分析摘要。
2. 问股：新问股、多轮追问、新建会话、加入自选。
3. 记录：历史记录列表、问股详情、报告记录详情、继续追问入口。
4. 我的：登录、注册、退出登录、后端地址配置、应用信息。

本版本不新增底部 Tab，不恢复大盘页。

## 6. 验证计划

执行完成后需要验证：

1. 未登录打开自选页，看到登录引导。
2. 未登录打开问股页，看到登录引导。
3. 未登录打开记录页，看到登录引导。
4. 登录后自选、问股、记录正常加载。
5. 问股发起新问题后生成 session。
6. 点击“新建会话”后，旧消息和 session 清空。
7. 在同一会话追问后，记录列表对应记录移动到顶部。
8. 记录详情点击“继续追问”后进入问股页，并继续沿用原 session。
9. 自选页展示最近一次问股或报告摘要。
10. 自选分析生成的记录类型显示为“报告”。
11. 旧 `analysis` 记录仍能打开。
12. 加入自选失败时不再误提示成功。
13. `SettingsScreen.tsx`、`MarketScreen.tsx` 删除后无残留引用。
14. API client 能显示后端返回的统一错误 message。
15. VS Code diagnostics 无报错。
16. 后端服务可启动。
17. 前端 TypeScript 检查或 Expo 编译检查无明显错误。
