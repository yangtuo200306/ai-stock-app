# AI Stock App 产品总路线

## 1. 文档定位

本文档用于记录 AI Stock App 在 v0.3 之后的产品总方向。

它是总计划，不替代每个版本的详细实施计划。后续每个版本仍需要单独制定小计划，例如 `v0.4-plan.md`、`v0.5-plan.md`。

## 2. 产品定位

AI Stock App 的定位是：

> 面向个人投资者的 AI 股票分析助手。

核心目标是帮助用户围绕 A 股完成：

1. 关注股票；
2. 提出问题；
3. 获取 AI 辅助分析；
4. 保存分析记录；
5. 后续复看和继续分析。

本项目不是专业交易终端，也不是完整投研工作台。

## 3. 当前阶段明确不做

在 UI/UX 大优化前，以下功能不纳入主线：

1. 持仓账户；
2. 交易记录；
3. 回测；
4. 多市场支持；
5. 预警推送；
6. 复杂 Agent 技能系统；
7. 专业量化平台能力。

用户系统需要做，但应在 UI/UX 大优化前完成最小版本，避免后续返工。

## 4. 长期底部导航方向

长期底部导航建议调整为：

```text
自选 | 问股 | 记录 | 我的
```

### 4.1 自选

职责：管理用户关注的股票。

包含：

1. 添加股票；
2. 删除股票；
3. 查看自选列表；
4. 从自选发起分析或问 AI；
5. 后续展示最近一次分析摘要。

不包含：

1. 持仓；
2. 成本价；
3. 盈亏；
4. 交易流水。

### 4.2 问股

职责：AI 股票分析助手主入口。

包含：

1. 输入股票问题；
2. 支持股票代码，后续支持股票简称；
3. 返回 AI 回答；
4. AI 失败时规则回退；
5. 问股结果可加入自选；
6. 后续支持最小多轮追问。

不包含：

1. 回测；
2. 专业策略系统；
3. 持仓建议。

### 4.3 记录

职责：保存用户触发过的所有分析行为。

记录不是只保存问股，也不是只保存正式报告。

记录应包含：

1. 问股记录；
2. 自选分析记录；
3. AI 对话记录；
4. 后续生成的报告记录。

基本逻辑：

```text
每次问股 -> 生成记录
自选点分析 -> 生成记录
生成报告 -> 生成记录
```

记录可以有不同类型：

```text
ask       问股
analysis  基础分析
report    报告
chat      多轮对话
```

报告是记录中的一种更正式的详情形态。用户普通提问不一定生成报告，但应该生成记录。

### 4.4 我的

职责：账号、设置和应用信息。

包含：

1. 注册；
2. 登录；
3. 退出登录；
4. 后端地址配置；
5. AI 服务状态；
6. 免责声明；
7. 版本信息。

## 5. 页面优化前需要完成的关键功能

UI/UX 大优化前，需要先完成会影响页面结构、导航、接口和数据归属的功能。

### 5.1 用户系统

用户系统会影响：

1. 我的页；
2. 所有接口请求方式；
3. 自选、记录、报告的数据归属；
4. 未登录态、登录态、退出登录态；
5. 后续页面权限和错误态。

因此不建议在 UI/UX 优化后再补用户系统。

### 5.2 记录体系

记录体系会影响：

1. 底部 Tab 是否从“报告”调整为“记录”；
2. 问股历史如何保存；
3. 自选分析如何保存；
4. 正式报告是否作为记录详情的一种类型；
5. 后续多轮追问和历史复看。

### 5.3 自选和问股联动

需要完成：

1. 问股结果加入自选；
2. 自选页快速问 AI；
3. 自选页后续展示最近分析摘要；
4. 自选数据绑定用户。

### 5.4 多轮追问最小版

如果问股页后续要像 AI 助手，就需要最小多轮追问能力。

最小目标：

1. 保存会话 id；
2. 保存多条消息；
3. 支持围绕同一只股票继续追问；
4. 不做复杂 Agent 技能系统；
5. 不做上下文压缩平台。

### 5.5 股票简称识别

股票简称识别会影响输入体验和错误提示。

最小目标：

1. 支持 A 股名称或简称识别；
2. 例如“平安银行怎么看”可识别为 `000001`；
3. 遇到歧义时给出提示；
4. 暂不扩展港股、美股。

### 5.6 API 错误结构统一

UI 优化会涉及错误态，因此需要先统一最小错误结构。

建议方向：

```text
{
  error_code,
  message
}
```

重点处理：

1. 不直接暴露 `llm_error`；
2. 区分输入错误、行情失败、AI 回退、服务异常；
3. 前端不长期依赖 FastAPI 默认 `detail` 字段。

### 5.7 AI 回答结构边界

UI 优化前需要明确 AI 回答是继续纯文本，还是逐步结构化。

建议方向：

1. 短期可以继续保存完整文本；
2. 前端按段落和模块展示；
3. 后续可扩展为结论、行情摘要、技术面、风险、后续观察点。

## 6. 推荐版本路线

### v0.4：用户边界 + 问股闭环 ✅ 已完成

目标：先把记录体系和用户归属边界搭起来。

已完成范围：

1. 预留默认 `user_id`；
2. 新增记录体系最小版；
3. 问股成功后写入记录；
4. 自选分析成功后写入记录；
5. 问股结果可加入自选；
6. 自选页可快速问 AI；
7. 底部 Tab 从"报告"调整为"记录"；
8. 隐藏 `llm_error`，新增 `ai_status`；
9. 问股页做最小滚动修复。

详见：`docs/releases/v0.4-summary.md`

### v0.5：最小用户系统 ✅ 已完成

目标：让 App 具备基础账号能力。

已完成范围：

1. 新增 `users` / `tokens` 表；
2. 完成注册、登录、退出登录和 `/api/auth/me`；
3. 使用 `hashlib.pbkdf2_hmac` + salt 保存密码哈希；
4. 使用 `uuid4` token 和 `Authorization: Bearer <token>`；
5. 后端自选、问股、分析任务、报告、记录均按真实用户隔离；
6. `stocks` 唯一约束调整为 `UNIQUE(user_id, code)`；
7. 前端新增 AuthContext、LoginScreen、MineStackNavigator 和统一 API client；
8. 我的页显示登录 / 未登录状态并支持退出登录；
9. 业务页面已接入 token 请求。

详见：`docs/releases/v0.5-summary.md`

### v0.6：AI 助手能力增强

目标：让 App 更像 AI 股票分析助手，而不是单次接口展示。

建议范围：

1. 股票简称识别；
2. 多轮追问最小版；
3. 技术指标增强；
4. AI 回答结构边界确定；
5. 问股历史详情增强。

### v0.7：功能定型

目标：在 UI/UX 大优化前完成产品功能定型。

建议范围：

1. 自选页展示最近一次分析摘要；
2. 记录详情页整理；
3. 报告作为记录详情的一种类型固定下来；
4. API 错误结构统一；
5. 清理调试字段；
6. 清理重复页面和重复逻辑；
7. 确认底部导航和页面职责。

### v0.8：UI/UX 集中优化

目标：从功能可用升级为体验好用。

建议范围：

1. 自选页优化；
2. 问股页优化；
3. 记录页优化；
4. 我的页优化；
5. 全局视觉统一；
6. 加载态、空状态、错误态统一；
7. 长回答阅读体验优化。

## 7. v0.8 之后只做小功能增强

v0.8 之后不再加入会推翻页面结构的大功能。

适合的小功能：

1. 快捷问题模板；
2. AI 回答复制；
3. 分享分析结果；
4. 历史搜索；
5. 自选排序；
6. 手动刷新行情；
7. 免责声明优化；
8. 最近查看股票。

继续排除的大功能：

1. 持仓；
2. 回测；
3. 多市场；
4. 预警推送；
5. 复杂 Agent 技能系统；
6. 专业投研工作台。

## 8. 总结

后续主线为：

```text
先定信息架构 -> 再补核心功能 -> 再做用户系统 -> 再增强 AI -> 再功能定型 -> 最后统一优化 UI
```

当前不追求马上美化页面，也不无限制加功能。

目标是在 UI/UX 大优化前，先让以下四个 Tab 的职责稳定：

```text
自选 | 问股 | 记录 | 我的
```
