# v0.5 最小用户系统详细计划

## 1. 版本目标

v0.5 的目标是让 AI Stock App 从“所有数据归属默认用户”升级为“具备基础账号能力的多用户系统”。

本版本重点不是 UI 精修，也不是复杂权限系统，而是先完成最小用户闭环：

1. 用户可注册账号。
2. 用户可登录获取 token。
3. 用户可退出登录。
4. 所有核心数据按真实用户隔离。
5. 未登录态有基本处理。

## 2. 本版本范围

### 2.1 做

1. 后端新增 `users` 表。
2. 后端新增 `tokens` 表。
3. 后端新增 `POST /api/auth/register` 注册接口。
4. 后端新增 `POST /api/auth/login` 登录接口。
5. 后端新增 `POST /api/auth/logout` 退出接口。
6. 后端新增 `GET /api/auth/me` 获取当前用户信息接口。
7. 后端新增鉴权依赖函数 `get_current_user_id`，从请求头 `Authorization: Bearer <token>` 解析用户。
8. 后端所有现有接口从 `DEFAULT_USER_ID` 改为 `get_current_user_id`。
9. 后端 `stocks` 表唯一约束从 `code UNIQUE` 改为 `(user_id, code)` 组合唯一。
10. 前端新增 `LoginScreen` 登录/注册页面。
11. 前端改造 `MineScreen`：未登录显示登录入口，已登录显示用户名和退出按钮。
12. 前端新增 `AuthContext` 管理登录状态和 token。
13. 前端新增统一 API client，自动带 token。
14. 前端所有页面从独立 fetch 改为统一 API client。
15. 未登录态处理：引导用户登录。

### 2.2 不做

1. 不做密码修改。
2. 不做密码重置。
3. 不做邮箱/手机验证。
4. 不做 OAuth 第三方登录。
5. 不做多设备登录管理。
6. 不做 token 过期自动刷新。
7. 不做用户头像、昵称修改。
8. 不做权限分级。
9. 不做 UI/UX 大优化。
10. 不删除 `DEFAULT_USER_ID` 常量，保留作为 fallback。

## 3. 后端设计

### 3.1 users 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | 用户 ID |
| `username` | TEXT NOT NULL UNIQUE | 用户名 |
| `password_hash` | TEXT NOT NULL | 密码哈希 |
| `password_salt` | TEXT NOT NULL | 密码盐值 |
| `created_at` | TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 3.2 tokens 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | Token ID |
| `user_id` | INTEGER NOT NULL | 关联用户 ID |
| `token` | TEXT NOT NULL UNIQUE | Token 值 |
| `created_at` | TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 3.3 密码加密方案

使用 Python 内置 `hashlib.pbkdf2_hmac`。

原则：

1. 不明文保存密码。
2. 每个用户使用独立 salt。
3. v0.5 不引入 bcrypt 等第三方依赖。

### 3.4 Token 方案

使用 `uuid4` 生成简单 token。

原则：

1. 登录和注册成功后返回 token。
2. 前端保存 token 到 AsyncStorage。
3. 后续接口通过 `Authorization: Bearer <token>` 传递。
4. 退出登录时服务端删除 token。
5. v0.5 不做 token 过期和刷新。

### 3.5 auth API

新增文件：

- `backend/app/api/auth.py`

#### POST /api/auth/register

请求体：

```json
{
  "username": "user_001",
  "password": "123456"
}
```

校验：

1. `username` 长度 3-20。
2. `username` 只允许字母、数字、下划线。
3. `password` 长度至少 6 位。
4. `username` 不能重复。

成功返回：

```json
{
  "message": "register success",
  "user_id": 1,
  "username": "user_001",
  "token": "uuid-string"
}
```

#### POST /api/auth/login

请求体：

```json
{
  "username": "user_001",
  "password": "123456"
}
```

成功返回：

```json
{
  "message": "login success",
  "user_id": 1,
  "username": "user_001",
  "token": "uuid-string"
}
```

失败返回：

```json
{
  "message": "invalid username or password",
  "error": "invalid_credentials"
}
```

#### POST /api/auth/logout

请求头带 token，服务端删除当前 token。

成功返回：

```json
{
  "message": "logout success"
}
```

#### GET /api/auth/me

请求头带 token，返回当前用户信息。

成功返回：

```json
{
  "user_id": 1,
  "username": "user_001",
  "created_at": "2026-06-24 12:00:00"
}
```

### 3.6 鉴权依赖函数

新增 `get_current_user_id`。

位置：

- 优先放在 `backend/app/database.py`，避免 v0.5 新增过多基础文件。

行为：

1. 读取 `Authorization` 请求头。
2. 校验格式必须为 `Bearer <token>`。
3. 从 `tokens` 表查询 token。
4. 找到则返回 `user_id`。
5. 找不到则返回 401。

### 3.7 现有接口改造

所有用户数据接口从 `DEFAULT_USER_ID` 改为 `Depends(get_current_user_id)`。

涉及文件：

| 文件 | 改造内容 |
|---|---|
| `backend/app/api/stocks.py` | 所有操作使用当前用户 |
| `backend/app/api/ask.py` | 写入记录时使用当前用户 |
| `backend/app/api/analysis.py` | 任务、报告、记录写入时使用当前用户 |
| `backend/app/api/reports.py` | 查询时使用当前用户 |
| `backend/app/api/records.py` | 查询时使用当前用户 |

### 3.8 stocks 表唯一约束改造

当前 `stocks.code` 是全局 `UNIQUE`，v0.5 需要改为 `(user_id, code)` 组合唯一。

计划：

1. 在 `database.py` 新增 `_migrate_stocks_table`。
2. 检测当前 `stocks` 表是否仍是旧唯一约束。
3. 如为旧结构，创建 `stocks_new`。
4. `stocks_new` 使用 `UNIQUE(user_id, code)`。
5. 复制旧数据。
6. 删除旧表。
7. 重命名新表为 `stocks`。

## 6. 验收标准

### 6.1 后端验收

1. 后端启动无报错。
2. 数据库自动创建 `users` 表。
3. 数据库自动创建 `tokens` 表。
4. `stocks` 表支持同一股票被不同用户添加。
5. `POST /api/auth/register` 可注册新用户并返回 token。
6. 重复注册同用户名返回明确错误。
7. `POST /api/auth/login` 可登录并返回 token。
8. 错误密码登录失败。
9. `GET /api/auth/me` 带 token 可返回当前用户。
10. `POST /api/auth/logout` 可退出并删除 token。
11. 退出后旧 token 不能继续访问受保护接口。
12. 不带 token 访问自选、问股、记录、报告接口返回 401。
13. 用户 A 和用户 B 的自选互不影响。
14. 用户 A 和用户 B 的记录互不影响。
15. 用户 A 和用户 B 的报告互不影响。

### 6.2 前端验收

1. 我的页未登录时显示登录/注册入口。
2. 可以注册新用户。
3. 可以登录已有用户。
4. 登录后我的页显示用户名。
5. 登录后可以添加自选。
6. 登录后可以问股。
7. 登录后可以查看记录。
8. 退出登录后回到未登录态。
9. 退出登录后无法继续访问需要登录的数据。
10. 切换不同用户后，自选和记录互相隔离。
