# v1.9.3 资讯/新闻集成 — 实施计划

> 基于创新模式讨论后制定，经审查模式修正。

---

## 核心目标

问股时自动抓取相关新闻，注入 Prompt 让 LLM 引用，同时在前端展示来源链接。分析报告底部也展示新闻链接列表。

---

## 一、新增模块：`backend/app/services/news_fetcher.py`

独立模块，不绑定任何流程，两个地方都能调。

```python
fetch_news(stock_code, stock_name) -> list[NewsItem]
```

- **数据源：** 新浪财经搜索（主）+ 东方财富（备），免费无 Key
- **解析方式：** requests + BeautifulSoup 解析 HTML
- **相关性过滤：** 标题必须包含股票代码、全称或简称之一
- **数量：** 取最近 3 条，按时间倒序
- **缓存：** 30 分钟 TTL（复用 `_quote_cache` 模式）
- **降级：** 两个源都失败 → 返回空列表，不影响主流程

```python
class NewsItem(BaseModel):
    title: str
    url: str
    source: str       # "新浪财经" / "东方财富"
    published_at: str  # 发布时间
```

## 二、后端改动

### 2.1 `llm_client.py` — Prompt 注入新闻

- `_build_prompt()` 新增 `news: list[dict]` 参数
- Prompt 中新增 `## 相关资讯` 区块
- `ask_llm()` 和 `ask_llm_stream()` 新增 `news` 参数，透传给 `_build_prompt()`
- System Prompt 增加一句：*"可以引用相关资讯中的数据来支撑你的分析，但需注明来源。"*

### 2.2 `ask.py` — 问股流程接入新闻

**非流式端点 (`POST /api/ask`)：**

第 107 行之后（`build_analysis_report` 之后、LLM 调用之前）：

```python
report = build_analysis_report(quote, technicals)
news = fetch_news(stock_code, report["stock_name"])  # ← 新增
```

- `AskResponse` 新增 `news: list[NewsItem]` 字段
- 调用 `ask_llm()` 时传入 `news=news`

**流式端点 (`POST /api/ask/stream`)：**

- `_build_report()` 之后、stream 开始之前，调用 `fetch_news()`
- `result_data` JSON 中加入 `news` 字段
- 调用 `ask_llm_stream()` 时传入 `news=news`

### 2.3 `analysis.py` — 分析流程接入新闻

第 97 行之后（`build_analysis_report` 之后、`INSERT INTO reports` 之前）：

```python
report = build_analysis_report(quote, technicals)
news = fetch_news(analysis.stock_code, report["stock_name"])  # ← 新增
```

- INSERT reports 时写入 `json.dumps(news)` 到 `news_json` 列

### 2.4 `reports.py` — 报告详情返回新闻

- `GET /api/reports/{id}` 返回中新增 `news` 字段，从 `news_json` 解析

### 2.5 `database.py` — 数据库迁移 v4

需要同步更新 3 处：

1. `CURRENT_SCHEMA_VERSION` → `"V20260625_004_add_news_json"`
2. `MIGRATIONS` 列表新增第 4 项
3. `_run_migration()` 新增 `elif` 分支

迁移 SQL（复用已有 `ensure_column()`）：

```python
ensure_column(connection, "reports", "news_json", "TEXT DEFAULT '[]'")
```

## 五、影响面分析

| 维度 | 检查结果 |
|------|---------|
| **数据链路** | `news_fetcher.py` → `ask.py`/`analysis.py` → `AskResponse`/`reports` → 前端渲染。每个环节都透传 `news` 列表 |
| **数据库** | `reports` 表新增 `news_json` 列，迁移 v4（含 `CURRENT_SCHEMA_VERSION`、`MIGRATIONS`、`_run_migration()` 三处更新） |
| **前后端类型** | 后端 `NewsItem` / `AskResponse.news` ↔ 前端 `NewsItem` / `AskResponse.news` / `Report.news`，类型同步 |
| **错误处理** | 新闻获取失败 → 返回空列表，不抛异常、不影响主流程。LLM 降级路径不受影响 |
| **配置** | 无需新增环境变量（两个新闻源都是免费公开接口） |
