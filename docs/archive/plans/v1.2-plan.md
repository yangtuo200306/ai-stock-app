# v1.2 自选页精修计划

> 基于 v1.1 问股页精修完成后的下一阶段页面精修。
> 对应总体方案阶段二（页面精修）的第二个页面。

---

## 设计目标

将自选页从"功能验证型"升级为"产品体验型"，聚焦日常使用频次最高的场景：**快速查看自选股状态、便捷添加/搜索/删除**。

## 改动原则

- 不改后端（阶段一已完成）
- 不改 Zustand store 架构
- 不改导航结构
- 不改主题系统

## 实施检查清单

### 1. 新增 ConfirmDialog.tsx

- **位置**：`mobile-app/src/components/ConfirmDialog.tsx`
- **功能**：通用确认弹窗
- **实现细节**：
  - 使用 React Native 内置 `Modal` 组件
  - 半透明黑色遮罩（`rgba(0,0,0,0.5)`）
  - 居中白色卡片（`colors.surface`），圆角 16
  - 标题（`typography.sectionTitle`）+ 描述（`typography.body`）
  - 底部两个按钮：取消（ghost 变体）+ 确认（danger 变体）
  - Props：`visible`, `title`, `description`, `confirmLabel`, `cancelLabel`, `onConfirm`, `onCancel`
- **复用性**：后续记录删除、退出登录等场景也可复用

### 2. 新增 WatchlistStockCard.tsx

- **位置**：`mobile-app/src/components/WatchlistStockCard.tsx`
- **功能**：从 WatchlistScreen 抽离股票卡片为独立组件
- **视觉升级**：
  - 左侧类型色条（4px 宽竖条，基于 `latest_record_type`：ask=蓝、report=青、analysis=紫、无记录=灰）
  - 代码用等宽字体（`fontFamily: 'monospace'`）+ 名称常规字体
  - 任务状态标签改为紧凑 Badge（圆角 6px，小字号）
  - 摘要区域最多 2 行，末尾省略号（`numberOfLines={2}`）
  - 底部操作行改为图标按钮（问 AI 图标 + 删除图标），替代全宽文字按钮
  - 删除按钮点击触发 `onDeleteRequest` 回调（由父组件控制确认弹窗）
- **Props**：
  ```typescript
  type WatchlistStockCardProps = {
    stock: Stock;
    taskStatus?: string | null;
    onPress: () => void;
    onAskAI: () => void;
    onDeleteRequest: () => void;
  };
  ```

### 3. 修改 watchlistStore.ts

- 新增 `searchQuery: string` 状态（初始值 `''`）
- 新增 `setSearchQuery(query: string)` action
- 新增 `getFilteredStocks(query: string)` 函数（非 Zustand getter，Zustand 无计算属性机制）
  - 过滤逻辑：匹配 `stock.code` 或 `stock.name`，大小写不敏感
  - 空查询时返回全部
- 其余逻辑不变

### 4. 重写 WatchlistScreen.tsx

#### 4.1 标题行
- 左侧："自选股" + `(N)` 数量标签（灰色小字）
- 右侧：⊕ 添加按钮（点击展开/收起添加表单）
- 移除旧版"自选股列表"标题和说明文案

#### 4.2 搜索框
- 位于标题行下方
- `TextInput` + 自定义清除按钮（×），兼容 Android（`clearButtonMode` 仅 iOS）
- placeholder："搜索自选股代码或名称"
- 输入时实时调用 `setSearchQuery`
- 搜索状态时显示过滤结果

#### 4.3 添加表单（可折叠）
- 默认收起
- 点击 ⊕ 按钮展开/收起
- 使用条件渲染（`showForm` state），不做动画
- 展开时显示：代码输入 + 名称输入 + 添加按钮
- 代码输入框增加即时格式校验：`normalizeAShareCode` 结果反馈（绿色 ✓ / 红色 ✗）
- 添加成功后自动收起表单并清空输入

#### 4.4 Pull-to-Refresh
- 使用 `FlatList` 的 `refreshing` + `onRefresh` 属性
- 下拉时调用 `loadStocks()` + `loadTaskStatuses()`
- 移除旧版"刷新列表"按钮

#### 4.5 删除确认弹窗
- 点击卡片删除图标 → 弹出 `ConfirmDialog`
- 文案："确定删除 {stockName}({stockCode})？"
- 确认后执行 `deleteStock`，取消则关闭弹窗

#### 4.6 空态/错误态美化
- **空态**：居中图标（星形/空列表图标）+ "暂无自选股" + "点击右上角 ⊕ 添加股票"
- **错误态**：居中图标 + "加载失败" + 重试按钮
- **加载态**：居中 `ActivityIndicator`

#### 4.7 使用 WatchlistStockCard
- `renderStockItem` 改为使用 `WatchlistStockCard` 组件
- 点击卡片 → `handleCreateAnalysis`
- 问 AI 按钮 → `handleAskAI`
- 删除请求 → 打开 ConfirmDialog

### 5. 验证

- `npx tsc --noEmit` 编译零错误
- 核心链路走通：登录 → 添加自选 → 查看列表 → 搜索过滤 → 删除（确认弹窗） → 下拉刷新
- 未登录态：显示 `LoginRequiredView`
- 空态/错误态/加载态展示正确

## 不做范围

- 不改后端接口（不增加行情价格字段）
- 不做股票搜索自动补全（需要后端股票索引数据）
- 不做拖动排序
- 不做 Swipe-to-Delete（React Native 需要额外原生依赖）
- 不修改导航类型（保持现有 any 兼容）
- 不修改主题系统
- 不修改导航结构
