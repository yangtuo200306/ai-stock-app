# v1.3 记录页精修计划

> 基于 v1.2 自选页精修完成后的下一阶段页面精修。
> 对应总体方案阶段二（页面精修）的第三个页面。

---

## 设计目标

将记录页从"功能验证型"升级为"产品体验型"，聚焦两个核心场景：**列表页快速浏览和搜索历史记录、详情页清晰阅读对话和指标**。

## 改动原则

- 不改后端
- 不改导航结构
- 不改主题系统
- 不加删除功能

## 实施检查清单

### 1. 新增 RecordCard.tsx

- **位置**：`mobile-app/src/components/RecordCard.tsx`
- **功能**：从 RecordListScreen 抽离记录卡片为独立组件
- **视觉风格**：与 WatchlistStockCard 保持一致的视觉语言
- **布局**：
  - 左侧类型色条（4px 宽竖条，基于 `record_type`：ask=蓝、report=青、analysis=紫）
  - 顶部行：类型 Badge（紧凑圆角标签）+ 时间（右对齐，灰色小字）
  - 股票名称 + 代码（代码用等宽字体 `fontFamily: 'monospace'`）
  - 标题（1 行，超出省略）
  - 摘要（2 行，`numberOfLines={2}`）
  - 底部 meta 行：评分 + 涨跌幅（带颜色），用圆点分隔符
- **触摸区域**：与 WatchlistStockCard 一致的 padding 和间距
- **Props**：
  ```typescript
  type RecordCardProps = {
    item: RecordItem;
    onPress: () => void;
  };
  ```

### 2. 修改 recordsStore.ts

- 新增 `searchQuery: string` 状态（初始值 `''`）
- 新增 `setSearchQuery(query: string)` action
- 新增 `getFilteredRecords(query: string)` 函数
  - 过滤逻辑：匹配 `stock_code`、`stock_name`、`title`、`summary`，大小写不敏感
  - 空查询时返回全部
- 其余逻辑不变

### 3. 重写 RecordListScreen.tsx

#### 3.1 标题行
- 左侧："记录" + `(N)` 数量标签（灰色小字）
- 风格与自选页标题行一致

#### 3.2 搜索框
- 位于标题行下方
- `TextInput` + 自定义清除按钮（×），兼容 Android
- placeholder："搜索记录"
- 输入时实时调用 `setSearchQuery`
- 搜索时显示过滤结果

#### 3.3 Pull-to-Refresh
- 使用 `FlatList` 的 `refreshing` + `onRefresh` 属性
- 下拉时调用 `fetchRecords()`
- 移除旧版"刷新"按钮（当前在空态中）

#### 3.4 使用 RecordCard
- `renderItem` 改为使用 `RecordCard` 组件
- 点击卡片 → `navigation.navigate('RecordDetail', { recordId: item.id })`

#### 3.5 空态区分
- **搜索无结果**："未找到匹配的记录" + 搜索关键词
- **真正无记录**："暂无记录" + "问股或自选分析后，这里会显示你的分析记录。"

#### 3.6 状态展示
- loading 态：居中 `ActivityIndicator`
- error 态：居中图标 + "记录加载失败" + 重试按钮

### 4. 修改 RecordDetailScreen.tsx

#### 4.1 类型标签改为色条风格
- 顶部增加 4px 类型色条（与 RecordCard 一致）
- 类型文字改为紧凑 Badge 风格

#### 4.2 股票代码加等宽字体
- `stock_code` 使用 `fontFamily: 'monospace'`
- 与自选页卡片风格统一

#### 4.3 指标区增加分区标题
- 指标区上方增加小标题"行情摘要"（`typography.label`，灰色）
- 与上方内容用间距分隔

#### 4.4 消息区增加分区标题
- 消息列表上方增加小标题"对话"（仅 `ask` 类型）
- 与上方内容用间距分隔

#### 4.5 底部按钮改为紧凑风格
- "继续追问"和"查看完整报告"按钮改为紧凑样式（参考 WatchlistStockCard 的图标按钮风格）
- 使用 `Pressable` + 主题色背景，替代全宽 `AppButton`

#### 4.6 不做
- 不改数据获取方式（仍用 `apiGet` 直接调用）
- 不加删除功能
- 不改导航

### 5. 验证

- `npx tsc --noEmit` 编译零错误
- 列表页：搜索过滤 → Pull-to-Refresh → 点击进详情
- 详情页：消息展示 → 指标展示 → 跳转问股/报告
- 空态/错误态/加载态展示正确

## 不做范围

- 不改后端接口
- 不加删除记录功能
- 不改导航结构
- 不改主题系统
- 不改数据获取方式（RecordDetail 仍用 apiGet）
- 不做分页（当前记录数较少）
