# AI Stock App v0.8 页面体验与公共 UI 基础计划

## 1. 版本目标

v0.8 目标：把 AI Stock App 从“功能验证页面”升级为“页面体验基本定型的小产品”。

核心方向：

1. 页面体验定型；
2. 公共 UI 基础；
3. 状态展示统一；
4. AI 长回答阅读体验优化；
5. 不引入复杂状态管理；
6. 不新增大功能。

对应路线文档：

- `docs/plans/v0.8-v1.0-roadmap.md`
- `docs/plans/product-roadmap.md`

## 2. 参考项目可借鉴点

参考项目是 Web 项目，不能直接复制到 React Native，但可以借鉴结构思想。

可借鉴：

1. 公共按钮组件思想：`stock-analysis-reference/apps/dsa-web/src/components/common/Button.tsx`
2. 公共卡片 / 分区卡片思想：
   - `stock-analysis-reference/apps/dsa-web/src/components/common/Card.tsx`
   - `stock-analysis-reference/apps/dsa-web/src/components/common/SectionCard.tsx`
3. 空状态 / 加载状态 / 错误提示组件思想：
   - `stock-analysis-reference/apps/dsa-web/src/components/common/EmptyState.tsx`
   - `stock-analysis-reference/apps/dsa-web/src/components/common/Loading.tsx`
   - `stock-analysis-reference/apps/dsa-web/src/components/common/InlineAlert.tsx`
   - `stock-analysis-reference/apps/dsa-web/src/components/dashboard/DashboardStateBlock.tsx`
4. 历史列表卡片的信息层级：`stock-analysis-reference/apps/dsa-web/src/components/history/HistoryListItem.tsx`
5. 自选 / 股票条目的信息密度控制：`stock-analysis-reference/apps/dsa-web/src/components/history/StockBarItem.tsx`
6. 主题 token 思想：`stock-analysis-reference/apps/dsa-web/src/index.css`

不借鉴：

1. 不引入 Tailwind；
2. 不引入 Web DOM 组件；
3. 不引入复杂主题系统；
4. 不引入参考项目的大量业务功能；
5. 不做桌面端 / Web 端迁移。

## 4. 公共 UI 组件职责

### 4.1 AppButton

路径：`mobile-app/src/components/AppButton.tsx`

职责：

1. 统一按钮样式；
2. 支持 `primary`、`secondary`、`danger`、`ghost` 等最小 variant；
3. 支持 loading 和 disabled；
4. 替换页面中重复的 `Pressable + Text` 按钮。

主要影响页面：

- `mobile-app/src/screens/AskScreen.tsx`
- `mobile-app/src/screens/WatchlistScreen.tsx`
- `mobile-app/src/screens/RecordListScreen.tsx`
- `mobile-app/src/screens/RecordDetailScreen.tsx`
- `mobile-app/src/screens/ReportDetailScreen.tsx`
- `mobile-app/src/screens/TaskStatusScreen.tsx`

### 4.2 AppCard

路径：`mobile-app/src/components/AppCard.tsx`

职责：

1. 统一白色卡片；
2. 统一圆角、内边距、最大宽度、背景色；
3. 支持普通卡片和列表卡片；
4. 替换各页面重复 `card` 样式。

### 4.3 StateView

路径：`mobile-app/src/components/StateView.tsx`

职责：

1. 统一 loading；
2. 统一 empty；
3. 统一 error；
4. 支持 action 按钮；

### 4.4 LoginRequiredView

职责：

1. 统一未登录提示；

- `mobile-app/src/screens/WatchlistScreen.tsx`

3. 优化长回答行高和阅读宽度；

受影响页面：

## 10. 验证方式

执行完成后建议验证：

1. TypeScript 编译通过；
2. VS Code diagnostics 无新增报错；
3. App 可启动；
4. 未登录打开自选、问股、记录、详情、任务、报告时显示统一登录引导；
5. 登录页登录 / 注册 UI 正常；
6. 自选页添加、刷新、问 AI、删除入口仍可用；
7. 问股页发送问题、新建会话、加入自选仍可用；
8. 记录列表可加载；
9. 记录详情可打开；
10. 记录详情继续追问仍可跳到问股页；
11. 报告详情可打开；
12. 任务状态页可刷新并查看报告；
13. 我的页后端地址保存和测试连接仍可用；
14. 页面视觉风格统一，无明显样式断层。

## 11. 学习重点

v0.8 适合学习：

1. 为什么要抽公共组件；
2. theme 常量和硬编码颜色的区别；
3. UI 组件与业务逻辑的边界；
4. 状态展示统一为什么比单纯美化更重要；
5. React Native 页面如何逐步从“功能可用”走向“产品可用”；
6. 为什么 v0.8 不急着引入状态管理库。
