# AI Stock App v1.0 版本计划：阶段一架构升级

## 1. 版本定位

v1.0 定位为：

```text
阶段一：全局架构升级 — 前端 Zustand 状态管理 + 后端配置/日志/错误码/数据库迁移
```

v0.9 用 DataRefreshContext 解决了"页面间需要刷新同步"的问题，但本质是补丁方案。
v1.0 分两个子阶段推进：1A 前端架构升级，1B 后端架构升级。

---

## 子阶段 1A：前端架构升级（Zustand 状态管理）

### 目标

引入 Zustand 替代 DataRefreshContext 版本号信号模式，用真正的共享状态替代补丁方案。

### 范围

- 引入 Zustand 状态管理库
- 创建 3 个 store：watchlistStore、recordsStore、askStore
- 迁移 4 个页面：WatchlistScreen、RecordListScreen、AskScreen、TaskStatusScreen
- 删除 DataRefreshContext 及其所有引用
- API 层增强：401 自动清 token
- 类型按领域拆分（5 个文件）
- 新增 resetAllStores 集中重置机制（登出/401 时一键清空）

### 保留不变

- AuthContext（认证状态保持 Context 模式）
- 所有 UI 样式
- 后端代码
- 功能范围

### 实施清单

## 子阶段 1B：后端架构升级

### 目标

把后端从"学习阶段的最小实现"升级到"生产级质量"：配置集中管理、日志可查、错误码统一、数据库迁移有版本记录。

### 范围

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1 | 配置管理 | 抽 `settings.py`，集中管理 10 项配置 |
| 2 | 日志系统 | 12 个文件添加日志，三级输出 + 文件轮转 |
| 3 | 错误码体系 | 13 个枚举 + 全局异常处理器，替换 17 处字符串 |
| 4 | 数据库迁移 | schema_version 表 + 3 个版本化迁移 |

### 不做

- 不改数据库引擎（继续 SQLite）
- 不改 API 路由结构
- 不加新接口

### 涉及文件

```
backend/app/
  config/
    __init__.py           → 新增
    settings.py           → 新增（pydantic-settings 集中管理 10 项配置）
  logging_config.py       → 新增（控制台 + 文件轮转三级输出）
  errors.py               → 新增（13 个 ErrorCode 枚举 + 辅助函数）
  error_handler.py        → 新增（全局异常处理器）
  main.py                 → 引入 settings/logging/error_handler
  database.py             → 版本化迁移（schema_version 表 + 3 个版本）
  api/
    ask.py                → 移除 os.environ.get，替换 error_code
    auth.py               → 替换 error_code + 日志
    analysis.py           → 替换 error_code + 日志 + 修复重复 key bug
    stocks.py             → 日志
    records.py            → 替换 error_code + 日志
    reports.py            → 替换 error_code + 日志
    market.py             → 替换 error_code + 日志
  services/
    llm_client.py         → 移除 load_dotenv，改用 settings
```
