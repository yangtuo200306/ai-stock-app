# v1.4 我的页精修计划

> 基于 v1.3 记录页精修完成后的下一阶段页面精修。
> 对应总体方案阶段二（页面精修）的第四个页面。

---

## 设计目标

将"我的"页面从"功能混合页"升级为"清爽的个人中心"，同时保留开发者配置能力但不干扰普通用户。

## 改动原则

- 不改后端
- 不改导航结构
- 不改主题系统
- 不改 AuthContext

## 涉及文件

| 操作 | 文件 |
|------|------|
| 修改 | `mobile-app/src/screens/MineScreen.tsx` |
| 修改 | `mobile-app/src/screens/LoginScreen.tsx` |

## 实施检查清单

### 1. 重写 MineScreen.tsx

#### 1.1 用户头像区

- 圆形头像占位（直径 64px），主题色背景
- 显示用户名首字母（大写），白色文字，字号 28
- 下方显示用户名（已登录）或"未登录"（未登录）
- 下方显示状态标签：
  - 已登录：绿色圆点 + "已登录"
  - 未登录：灰色圆点 + "未登录"

#### 1.2 账号操作区

- 使用紧凑 `Pressable` 风格（与 WatchlistStockCard 按钮一致）
- 已登录：红色背景 "退出登录" 按钮
- 未登录：主题色背景 "登录 / 注册" 按钮

#### 1.3 应用信息区

- 从 `package.json` 读取 `version` 字段（`require('../../package.json').version`）
- 展示：
  - 应用名称 "AI Stock"
  - 版本号 "v{version}"
  - 简短描述

#### 1.4 开发者设置（可折叠）

- 使用 `useState<boolean>` 控制展开/收起
- 标题行：`▶ 开发者设置`（展开时变为 `▼`）
- 展开后保持现有逻辑不变：
  - 后端地址输入框
  - 保存地址按钮
  - 测试连接按钮
  - 操作反馈提示
- 保留 `handleSave`、`handleTestConnection`、`backendUrl`、`savedBackendUrl`、`message` 等状态和逻辑

#### 1.5 样式调整

- 移除旧版 `sectionTitle`、`sectionBody`、`placeholder`、`usernameText` 等样式
- 新增头像、状态标签、折叠面板相关样式
- 保持与自选页/记录页一致的 padding 和间距

### 2. 修改 LoginScreen.tsx

- 输入框之间增加视觉分隔（`borderBottom` 或间距调整）
- 错误提示改为带红色背景的 Badge 风格（`backgroundColor: colors.danger + 0.1` 透明度）
- 切换模式按钮文案更清晰
- 按钮风格与 MineScreen 的紧凑按钮保持一致

### 3. 验证

- `npx tsc --noEmit` 编译零错误
- 未登录态：显示头像占位 + "未登录" + 登录按钮
- 已登录态：显示头像占位 + 用户名 + 退出按钮
- 开发者设置默认收起，点击展开，再次点击收起
- 后端地址保存 + 测试连接功能正常
- 版本号正确显示（非硬编码）

## 不做范围

- 不改后端
- 不改 AuthContext
- 不改导航结构
- 不添加使用统计
- 不添加主题切换
- 不添加密码修改
- 不添加用户头像上传
