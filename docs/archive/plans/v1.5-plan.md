# v1.5 公共组件收口计划

> 基于 v1.4 我的页精修完成后的公共组件一致性收口。
> 对应总体方案阶段二（页面精修）的收尾步骤。

---

## 设计目标

统一公共组件的风格一致性，消除新页面与旧组件之间的视觉差异，为后续功能开发提供统一的组件基础。

## 改动原则

- 不改页面代码（只改组件）
- 不改主题系统
- 不改组件 Props 签名（AppButton 除外，新增可选 prop）

## 涉及文件

| 操作 | 文件 |
|------|------|
| 修改 | `mobile-app/src/components/AppButton.tsx` |
| 修改 | `mobile-app/src/components/StateView.tsx` |
| 修改 | `mobile-app/src/components/RecordCard.tsx` |

## 实施检查清单

### 1. AppButton 新增 compact 变体

- 新增可选 prop：`size?: 'default' | 'compact'`
- `compact` 尺寸：`minHeight: 36`，`borderRadius: 10`，`paddingVertical: spacing.sm`
- `default` 尺寸保持不变（`minHeight: 42`，`borderRadius: 12`）
- 现有所有调用方不受影响（默认 `size='default'`）

### 2. StateView 使用 compact AppButton

- `StateView` 内部调用 `AppButton` 时传入 `size="compact"`
- 空态/错误态中的操作按钮更紧凑，与页面风格一致

### 3. RecordCard 摘要颜色统一

- `summaryText` 颜色从 `colors.textMuted` 改为 `colors.textSecondary`
- 与 WatchlistStockCard 的摘要颜色一致

### 4. 验证

- `npx tsc --noEmit` 编译零错误
- AppButton 默认行为不变（所有旧页面不受影响）
- StateView 按钮变紧凑
- RecordCard 摘要颜色变深

## 不做范围

- 不修改页面代码
- 不修改主题系统
- 不删除任何组件
- 不修改 WatchlistStockCard
- 不修改其他组件
