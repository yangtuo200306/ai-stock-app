# v0.3 AI 问股可用版总结

## 1. 版本目标

v0.3 的目标是让 AI Stock App 从“规则型基础分析”升级为“更可用的 AI 问股助手”。

本版本重点完成两件事：

1. 提升行情稳定性：在 `efinance` 失败时尝试新浪直连 fallback。
2. 提升问股可用性：接入火山方舟 DeepSeek Endpoint，让问股返回 AI 风格回答。

## 2. 阶段 1：efinance + 新浪 fallback

已完成内容：

1. 保留 `efinance` 作为主行情源。
2. 新增 `requests` 依赖。
3. 新增新浪实时行情 fallback。
4. 新增新浪历史 K 线 fallback。
5. 新增 A 股 6 位代码到新浪 symbol 的转换逻辑。
6. `get_stock_quote(...)` 主源失败后尝试新浪 fallback。
7. `get_stock_history(...)` 主源失败后尝试新浪 fallback。
8. 两个源都失败时返回包含主源和备用源原因的聚合错误。
9. fallback 成功时可返回 `source = "sina"`。

涉及文件：

- `backend/requirements.txt`
- `backend/app/services/market_data.py`

## 3. 阶段 2：AI 问股最小可用版

已完成内容：

1. 新增 `backend/app/services/llm_client.py`。
2. 新增 `python-dotenv` 依赖。
3. 后端支持从 `.env` 读取大模型配置。
4. 问股接口支持 `question` 字段。
5. 后端可从用户问题中提取 6 位股票代码。
6. 后端继续复用真实行情、历史 K 线、技术指标和规则分析。
7. 后端构造大模型上下文，调用火山方舟 DeepSeek Endpoint。
8. 大模型成功时返回 `answer_type = "ai"`。
9. 大模型失败时回退规则回答。
10. 前端问股页从“股票代码输入”升级为“股票问题输入”。
11. 前端可展示 AI 回答、行情摘要、技术指标和风险提示。

涉及文件：

- `backend/app/services/llm_client.py`
- `backend/app/api/ask.py`
- `mobile-app/src/screens/AskScreen.tsx`
- `mobile-app/src/types/index.ts`
- `backend/requirements.txt`

## 4. 大模型配置

本版本使用 `.env` 保存本地大模型配置。

配置项：

```text
LLM_API_KEY=本地火山平台 API Key
LLM_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
LLM_MODEL=火山方舟 Endpoint ID
```

注意：

- `.env` 已被 `.gitignore` 忽略。
- API Key 不提交到 Git。
- 火山平台调用需要使用 Endpoint ID，例如 `ep-...`，不是模型商品名。

## 5. 验收结果

### 后端接口验收

已验证：

1. `GET /api/market/quote/abc` 返回明确的 6 位 A 股代码错误。

## 6. 已知问题

本版本新增和确认的问题已记录到 `docs/code-review-notes.md`。

重点包括：

1. 新浪 fallback 依赖非官方接口。
2. fallback 路径缺少强制模拟主源失败的自动化验证。
3. 新浪返回文本解析较脆弱。
4. fallback 聚合错误仍是纯文本。
5. `requests` 和 `python-dotenv` 等依赖暂未固定版本。
6. 火山方舟调用需要使用 Endpoint ID。
7. `llm_error` 当前仍是开发期调试字段。
8. 大模型调用依赖第三方平台稳定性。
9. Prompt 当前直接拼字符串。
10. AI 回答暂不保存历史。
11. 移动端页面整体模板化且视觉体验不足。

## 7. 后续建议

建议后续优先考虑：

1. 单独规划 UI/UX 优化阶段。
2. 重构问股页长回答展示体验。
3. 评估是否移除或改造 `llm_error` 调试字段。
4. 补充更清晰的大模型错误日志。
5. 后续考虑问股历史、多轮问答或更丰富的技术指标。

## 8. 版本结论

v0.3 已完成“行情更稳 + AI 问股可用”的主要目标。

当前 APP 已经可以基于真实行情、技术指标和规则分析上下文，调用大模型生成自然语言个股分析回答。
