# v0.6 AI 助手能力增强总结

## 1. 版本定位

v0.6 的目标是让 AI Stock App 从“单次问股接口展示”升级为“最小 AI 股票助手”。

本版本重点是 AI 助手能力增强，采用“两阶段实现、一个版本验收”的方式完成：

1. 阶段 1：AI 助手底座。
2. 阶段 2：AI 助手增强。

完成后，App 已支持股票简称识别、最小多轮追问、技术指标增强、AI 回答边界明确和问股历史详情增强。

## 2. 已完成内容

### 2.1 AI 助手会话底座

1. 新增 `ask_sessions` 表，用于保存一次问股会话。
2. 新增 `ask_messages` 表，用于保存会话中的多轮 user / assistant 消息。
3. `records` 表新增 `session_id` 字段，用于将问股记录关联到会话。
4. `/api/ask` 支持 `session_id`。
5. 首次问股会创建新会话。
6. 继续追问会复用同一个会话。
7. 同一会话固定绑定一只股票。
8. 追问时如果用户输入另一只股票，会提示新开问题。
9. 会话查询使用 `session_id + user_id`，防止跨用户访问。

### 2.2 问股记录策略调整

1. 新建问股会话时新增一条 `records`。
2. 继续追问时不新增 records，而是更新同一条会话记录。
3. records 摘要会更新为最近一次回答摘要。
4. records 的 `metadata_json` 已保存最新行情、模型、`answer_type` 和 `ai_status`。
5. 记录列表继续展示会话摘要，避免多轮追问刷出多条记录。

### 2.3 记录详情增强

1. `GET /api/records` 已返回 `session_id`。
2. `GET /api/records/{record_id}` 在有 `session_id` 时返回 `messages`。
3. 记录详情页可以展示完整多轮问答。
4. 没有 `messages` 的旧问股记录仍兼容单条 question / answer 展示。
5. 记录详情查询会按 `user_id` 过滤，保证用户隔离。

### 2.4 AI 上下文增强

1. `ask_llm` 支持 `conversation_messages` 入参。
2. 追问时只传最近 4 条消息，避免 prompt 过长。
3. prompt 已明确当前会话股票边界。
4. prompt 要求 AI 先回答当前问题，再补充技术指标分析。
5. prompt 保留“不提供确定性买卖指令”和“仅供学习参考，不构成投资建议”的边界。

### 2.5 股票简称识别

1. 新增 `backend/app/services/stock_resolver.py`。
2. 股票识别顺序为：

```text
明确 stock_code 参数
-> question 中 6 位代码
-> question 中内置股票名称
-> 失败提示用户输入 6 位 A 股代码或已支持名称
```

3. 当前支持少量常用 A 股名称：

```text
平安银行 -> 000001
万科A -> 000002
贵州茅台 -> 600519
招商银行 -> 600036
宁德时代 -> 300750
比亚迪 -> 002594
五粮液 -> 000858
中国平安 -> 601318
工商银行 -> 601398
东方财富 -> 300059
```

4. 暂不做全市场搜索、拼音、模糊匹配、港股、美股和复杂歧义处理。

### 2.6 技术指标增强

1. 新增 RSI 指标计算：`rsi6`、`rsi12`。
2. 新增成交量变化计算：`volume_ratio`、`volume_signal`。
3. `volume_ratio` 使用最近一日成交量 / 前 5 日平均成交量。
4. 历史数据不足 6 条时，成交量比值返回默认值 `0.0`。
5. 成交量信号规则：

```text
volume_ratio >= 1.5 -> 放量
volume_ratio <= 0.7 -> 缩量
其他 -> 平稳
```

6. 报告构建已将 RSI 和成交量写入 `indicators`。
7. `score_reasons` 已加入 RSI / 成交量相关原因。
8. `risks` 已加入 RSI 过热、RSI 偏弱和放量下跌等提示。
9. RSI 偏高会轻微扣分。
10. RSI 偏低不直接加分，只作为风险或状态提示。

### 2.7 前端问股页增强

1. `AskScreen` 从单个 `result` 调整为最小多轮消息展示。
2. 新增 `sessionId` 状态，用于保存当前会话 ID。
3. 新增 `messages` 状态，用于展示多轮聊天。
4. 新增 `latestResult` 状态，用于展示最近一次结构化分析数据。
5. 用户继续追问时会自动携带当前 `session_id`。
6. 问股页已展示：
   - 当前价；
   - 涨跌幅；
   - 趋势；
   - 建议；
   - 评分；
   - MA5 / MA10 / MA20；
   - RSI(6) / RSI(12)；
   - 成交量信号和成交量比值；
   - 风险提示。
7. 保留“加入自选”按钮。

### 2.8 报告详情兼容新增指标

1. `ReportIndicators` 类型已扩展 RSI 和成交量字段。
2. `ReportDetailScreen` 已展示 RSI 区域。
3. `ReportDetailScreen` 已展示成交量区域。
4. 自选分析和报告详情继续兼容原有字段。

## 3. 审查和修正结果

v0.6 首次审查发现 6 个偏差，均已修复并复审通过：

1. `stock_code` 参数优先级已修正为 `stock_code -> question`。
2. 追问时已拦截不同股票。
3. `ask_sessions.summary` 追问时已更新。
4. records metadata 已写入 `answer_type` 和 `ai_status`。
5. RSI 偏低直接加分逻辑已移除。
6. 成交量比值已要求至少 6 条历史数据。

复审结论：

```text
实施与计划完全相符
```

VS Code diagnostics：无报错。

## 5. 本版本不做内容

1. 不做全市场股票搜索。
2. 不做股票简称模糊匹配。
3. 不做拼音识别。
4. 不做港股、美股。
5. 不做 MACD 完整实现。
6. 不做复杂 Agent 技能系统。
7. 不做工具调用型 AI。
8. 不做流式输出。
9. 不做长上下文压缩。
10. 不做消息删除、编辑、重试。
11. 不做 API 错误结构统一。
12. 不做大规模 UI/UX 美化。
13. 不做记录模块完整重构。
14. 不做从记录详情继续追问。
15. 不做问股页历史会话恢复。

## 6. 学习重点

本版本重点学习了：

1. 为什么聊天系统需要 session 和 message 两层结构。
2. `ask_sessions` 保存会话概要，`ask_messages` 保存每条消息。
3. 为什么 records 更适合做历史目录，而不是聊天明细表。
4. 问股页为什么既需要 `messages`，又需要 `latestResult`。
5. 后端如何通过 `session_id` 区分新问股和继续追问。
6. 为什么会话查询必须带 `user_id`。
7. AI 上下文为什么只传最近几轮，不能无限传。
8. 股票简称识别为什么先做小词典，而不是全市场搜索。
9. RSI 的基本含义和阶段性简化实现。
10. 成交量变化如何辅助判断市场参与度。
11. API 响应结构为什么需要逐步建立边界。
12. 功能闭环和 UI/UX 打磨为什么适合拆阶段推进。

## 7. 已记录问题

v0.6 学习、体验和审查中发现的问题已记录到：

```text
docs/code-review-notes.md
```

新增重点问题包括：

1. Windows 命令行验收中文 JSON 容易乱码。
2. StopCommand 停止后台 uvicorn 失败。
3. AskScreen 本地 user message id 使用 `Date.now()`。
4. `ask.py` 会话、消息和记录逻辑集中在一个文件。
5. records 追问更新后不会按最近追问时间排序。
6. 股票简称识别当前只支持内置小词典。
7. 股票名称匹配使用简单 substring 判断。
8. RSI 计算使用简化版公式。
9. 成交量比值规则较简单。
10. 不同行情源成交量字段单位可能不完全一致。
11. AskScreen 多轮消息只保存在当前页面内存。
12. AskScreen 没有显式“新建会话”按钮。
13. RecordDetailScreen 只能查看历史，不能继续追问。
14. 未登录态下业务页面操作提示不够友好。
15. v0.6 问股页仍偏功能验证型 UI。

## 8. 当前产品状态

完成 v0.6 后，AI Stock App 已具备：

```text
账号系统
自选股
AI 问股
最小多轮追问
股票简称识别
技术指标增强
问股历史详情
记录系统
```

当前产品已经从“多用户最小闭环 App”升级为“具备 AI 股票助手雏形的学习型 App”。

核心体验已经可用，但交互仍偏功能验证型。
