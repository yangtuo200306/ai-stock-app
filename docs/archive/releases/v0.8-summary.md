# v0.8 页面体验与公共 UI 基础总结

## 1. 版本定位

v0.8 的目标是把 AI Stock App 从“功能验证页面”升级为“页面体验基本定型的小产品”。

本版本重点不是单纯美化，而是完成三件事：

1. 页面体验定型；
2. 公共 UI 基础；
3. 状态展示统一。

本版本延续 v0.7 已定型的 `自选 | 问股 | 记录 | 我的` 四个 Tab，不新增大功能，不引入复杂状态管理，不调整后端接口契约。

## 2. 已完成内容

### 2.1 新增公共 UI 组件

新增目录：

```text
mobile-app/src/components/
```

新增组件：

1. `AppButton.tsx`
   - 统一按钮样式。
   - 支持 `primary`、`secondary`、`danger`、`ghost`。
   - 支持 `loading` 和 `disabled`。

2. `AppCard.tsx`
   - 统一卡片背景、边框、圆角和 padding。
   - 用于页面主卡片、列表卡片和信息分区。

3. `StateView.tsx`
   - 统一 loading、empty、error 状态展示。
   - 支持 action 按钮。

4. `LoginRequiredView.tsx`
   - 统一未登录引导。
   - 跳转逻辑由页面传入，避免组件强绑定导航类型。

5. `MessageBubble.tsx`
   - 统一问股页和记录详情页的消息气泡。
   - 区分“我”“AI 回答”“规则回退回答”。
   - 优化长回答行高。

6. `MetricRow.tsx`
   - 统一指标行展示。
   - 支持 label / value 和 value 颜色。
   - 用于问股结果、记录详情、报告详情。

### 2.12 鉴权请求守卫和本地状态健壮性补齐

审查阶段发现并修复：

1. 部分业务页面虽然显示了统一未登录引导，但 effect 仍可能在未登录或鉴权恢复期间提前请求受保护接口。
2. `WatchlistScreen` 中 `stockTaskIds` 本地 JSON 损坏时，部分操作仍可能解析失败。

修复结果：

1. `WatchlistScreen`、`RecordListScreen`、`RecordDetailScreen`、`TaskStatusScreen` 的数据加载均增加 `authLoading || !isLoggedIn` 前置守卫。
2. 对应 effect 依赖已补充登录状态，登录状态变化后可重新触发加载。
3. `WatchlistScreen` 新增文件内 `readTaskIds()`，统一读取、解析和损坏清理 `TASK_IDS_KEY`。
4. `JSON.parse` 现在只存在于 `readTaskIds()` 内，并已被 try/catch 包裹。

## 6. 学习重点

本版本重点学习了：

1. 公共组件不是为了少写代码，而是为了统一体验。
2. theme 常量不是单纯美化，而是减少风格漂移。
3. 展示工具函数适合放文案、颜色、格式化，不适合放业务流程。
4. 页面应该负责业务流程，公共组件负责展示规范。
5. 复杂页面应最后改，先让公共基础层稳定。
6. `StateView` 统一 loading、empty、error，比单页美化更重要。
7. `LoginRequiredView` 统一未登录引导，但 effect 请求也必须配合鉴权守卫。
8. React Native 页面可以通过“小组件 + 小 theme + 小 utils”逐步产品化。
9. 不要把 v0.9 的状态管理问题塞进 v0.8，否则版本边界会失控。
10. 审查不是只看 TypeScript 是否通过，还要看计划是否被完整遵守。

## 7. 已记录问题

v0.8 学习、体验和审查中发现的问题已记录到：

```text
docs/code-review-notes.md
```

新增重点问题包括：

1. `MineScreen` 版本展示仍是页面硬编码。
2. `TaskStatusScreen` 仍需手动刷新任务状态。
3. 自选页任务状态依赖本地 `AsyncStorage` 映射。
4. 自选股删除缺少二次确认。
5. 问股页没有自动滚动到底部。
6. 问股失败后缺少重试上一条问题入口。
7. 加入自选后不会自动同步刷新自选页。
8. `AskScreen` 仍使用 `any` 导航类型。
9. 公共组件还没有 `compact / size` 变体。
10. `MessageBubble` 不支持复制和 Markdown 渲染。
11. A 股代码标准化仍只覆盖 6 位代码和常见前后缀。
12. 公共组件缺少单元测试或快照测试。
13. `MetricRow` 对长 value 的布局承载仍需实机观察。
14. `ReportDetailScreen` 的路由参数类型仍未集中复用。
15. 底层状态管理仍未解决，留到 v0.9。
