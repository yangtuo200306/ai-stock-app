# v0.4 用户边界 + 问股闭环总结

## 1. 版本目标

v0.4 的目标是让 AI Stock App 从"能问股、能看报告"升级为"用户行为可归属、问股和自选形成闭环"。

本版本重点不是 UI 精修，也不做真正注册登录，而是先搭好三个基础边界：

1. 所有核心数据预留默认 `user_id`。
2. 新增最小记录体系，承接问股、自选分析和报告。
3. 打通问股、自选、记录三个模块之间的基础联动。

## 2. 数据库变更

### 2.1 新增 DEFAULT_USER_ID

- 位置：`backend/app/database.py`
- 值：`"default_user"`
- 作用：v0.4 所有数据默认归属该用户，v0.5 做登录后替换为真实用户 ID。

### 2.2 旧表增加 user_id

| 表 | 新增字段 | 默认值 |
|---|---|---|
| `stocks` | `user_id TEXT NOT NULL DEFAULT 'default_user'` | `default_user` |
| `analysis_tasks` | `user_id TEXT NOT NULL DEFAULT 'default_user'` | `default_user` |
| `reports` | `user_id TEXT NOT NULL DEFAULT 'default_user'` | `default_user` |

### 2.3 新增 records 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | 记录 ID |
| `user_id` | TEXT NOT NULL | 用户 ID |
| `record_type` | TEXT NOT NULL | 记录类型：`ask` / `analysis` |
| `stock_code` | TEXT NOT NULL | 股票代码 |
| `stock_name` | TEXT NOT NULL | 股票名称 |
| `title` | TEXT NOT NULL | 记录标题 |
| `summary` | TEXT NOT NULL | 列表摘要 |
| `question` | TEXT | 问股问题 |
| `answer` | TEXT | 问股回答 |
| `answer_type` | TEXT | `ai` / `rule` |
| `report_id` | INTEGER | 关联报告 ID |
| `metadata_json` | TEXT NOT NULL | 附加信息 JSON |
| `created_at` | TEXT NOT NULL | 创建时间 |

## 3. 后端接口变更

### 3.1 新增接口

| 接口 | 作用 |
|---|---|
| `GET /api/records` | 获取记录列表，按 `user_id` 过滤 |
| `GET /api/records/{record_id}` | 获取记录详情，按 `user_id` 过滤 |

### 3.2 改造接口

| 接口 | 变更 |
|---|---|
| `GET /api/stocks` | 增加 `user_id` 过滤 |
| `POST /api/stocks` | 写入 `user_id` |
| `DELETE /api/stocks/{code}` | 增加 `user_id` 条件 |
| `POST /api/ask` | 移除 `llm_error`、新增 `ai_status`、成功后写入 `ask` 记录 |
| `POST /api/analysis` | 任务/报告写入 `user_id`、成功后写入 `analysis` 记录 |
| `GET /api/analysis/{task_id}` | 增加 `user_id` 过滤 |
| `GET /api/reports` | 增加 `user_id` 过滤 |
| `GET /api/reports/{report_id}` | 增加 `user_id` 过滤 |

## 4. 移动端变更

### 4.1 导航调整

- 底部 Tab 从"报告"改为"记录"。
- 新增 `RecordStackNavigator`，替换原 `ReportStackNavigator`。
- 记录模块 Stack：`RecordList -> RecordDetail -> ReportDetail`。

### 4.2 新增页面

| 页面 | 接口 | 作用 |
|---|---|---|
| `RecordListScreen` | `GET /api/records` | 记录列表页 |
| `RecordDetailScreen` | `GET /api/records/{record_id}` | 记录详情页 |

### 4.3 页面改造

| 页面 | 变更 |
|---|---|
| `AskScreen` | 外层改为 `ScrollView`、支持 `initialQuestion` 参数、新增"加入自选"按钮、不展示 `llm_error` |
| `WatchlistScreen` | 每只自选股新增"问 AI"按钮，点击跳转问股 Tab 并带入默认问题 |

### 4.4 删除页面

| 页面 | 原因 |
|---|---|
| `ReportHistoryScreen` | 被 `RecordListScreen` 替代 |

## 5. 类型变更

## 7. 新增问题记录

本版本新增 14 个问题/隐患，已追加到 `docs/code-review-notes.md`（问题 52-65）。

重点包括：

1. `records.metadata_json` 仍然是松散 JSON 字段
2. `records.py` 对 `metadata_json` 缺少异常保护
3. `stocks.code` 仍保留全局 UNIQUE 约束
4. `ask.py` 捕获 `LlmError` 后完全吞掉错误
5. `answer_type` 和 `ai_status` 当前语义有重叠
6. `analysis.py` 写报告和写记录不是显式同一个事务边界
7. 问股记录写入失败会影响问股接口返回
8. `AskScreen` 加入自选失败时仍提示"已加入自选"
9. `WatchlistScreen` 使用 `useNavigation<any>()` 进行跨 Tab 跳转

## 8. 版本结论

v0.4 已完成"用户边界 + 问股闭环"的主要目标。

当前 APP 已完成：

- 所有核心数据预留默认 `user_id`
- 最小记录体系，问股和自选分析自动写入记录
- 底部 Tab 调整为"自选 | 问股 | 记录 | 我的"
- 问股结果可加入自选
- 自选页可快速问 AI
- `llm_error` 已隐藏，新增 `ai_status` 替代

v0.4 交付范围完成，准备进入 v0.5 最小用户系统。
