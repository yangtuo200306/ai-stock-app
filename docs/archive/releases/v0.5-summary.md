# v0.5 最小用户系统总结

## 1. 版本定位

v0.5 的目标是让 AI Stock App 从“默认用户数据归属”升级为“具备基础账号能力的多用户系统”。

本版本重点是最小用户闭环，不做复杂权限系统，也不做完整个人中心。

## 2. 已完成内容

### 2.1 后端账号能力

1. 新增 `users` 表。
2. 新增 `tokens` 表。
3. 新增注册接口：`POST /api/auth/register`。
4. 新增登录接口：`POST /api/auth/login`。
5. 新增退出接口：`POST /api/auth/logout`。
6. 新增当前用户接口：`GET /api/auth/me`。
7. 使用 `hashlib.pbkdf2_hmac` + salt 保存密码哈希。
8. 使用 `uuid4` 生成最小 token。
9. 新增 `get_current_user_id`，从 `Authorization: Bearer <token>` 中解析当前用户。

### 2.2 用户数据隔离

以下接口已从 `DEFAULT_USER_ID` 改为真实用户 token 归属：

1. 自选股：`/api/stocks`
2. 问股：`/api/ask`
3. 分析任务：`/api/analysis`
4. 报告：`/api/reports`
5. 记录：`/api/records`

用户 A 和用户 B 的自选数据已通过接口验收确认互不影响。

### 2.3 数据库结构调整

1. `stocks` 表唯一约束从全局 `code UNIQUE` 调整为 `UNIQUE(user_id, code)`。
2. 新增 `_migrate_stocks_table`，用于迁移旧表结构。
3. 修正迁移判断逻辑：通过 `PRAGMA index_info(...)` 检查唯一索引字段，而不是只检查是否存在唯一索引。

### 2.4 前端账号能力

1. 新增统一 API client：`mobile-app/src/api/client.ts`。
2. 新增认证上下文：`mobile-app/src/contexts/AuthContext.tsx`。
3. 新增登录 / 注册页面：`LoginScreen.tsx`。
4. 新增“我的”模块 Stack：`MineStackNavigator.tsx`。
5. `AppNavigator` 已包裹 `AuthProvider`。
6. `MineScreen` 已支持未登录 / 已登录状态展示。
7. 登录成功后保存 token 到 AsyncStorage。
8. App 启动时会通过 `/api/auth/me` 恢复登录状态。
9. 退出登录会清除本地 token。

### 2.5 前端业务页面接入 token

以下页面已改用统一 API client：

1. `WatchlistScreen`
2. `AskScreen`
3. `RecordListScreen`
4. `RecordDetailScreen`
5. `TaskStatusScreen`
6. `ReportDetailScreen`

普通业务接口现在会自动携带：

```text
Authorization: Bearer <token>
```

## 3. 验收结果

### 3.1 后端验收

已通过：

1. 注册新用户，返回 token。
2. 重复注册，返回 `username_exists`。
3. 登录正确密码，返回 token。
4. 登录错误密码，返回 `invalid_credentials`。
5. `GET /api/auth/me` 返回当前用户信息。
6. 无 token 访问受保护接口，返回 401。
7. 退出登录返回 `logout success`。
8. 退出后旧 token 访问返回 `invalid token`。
9. 用户 A 添加自选后可见。
10. 用户 B 自选为空，确认用户隔离有效。

### 3.2 审查修正

v0.5 首次审查发现偏差后，已完成修正：

1. `logout` 从删除用户所有 token 改为只删除当前 token。
2. `stocks` 表迁移逻辑改为检查唯一索引字段。
3. `TaskStatusScreen` 改用 `apiGet`。
4. `ReportDetailScreen` 改用 `apiGet`。
5. `AskScreen` 清理重复后端地址读取逻辑。
6. `AuthContext` 中认证流程相关独立 `fetch` 保留为例外。

修正项复审结果：实施与修正计划完全相符。

## 4. 本版本不做内容

1. 不做密码修改。
2. 不做密码重置。
3. 不做邮箱或手机号验证。
4. 不做第三方登录。
5. 不做 token 过期和刷新。
6. 不做多设备登录管理。
7. 不做头像、昵称等完整个人资料。
8. 不做权限分级。
9. 不做 UI/UX 大优化。
10. 不迁移旧 `default_user` 数据到真实用户。

## 5. 学习重点

本版本重点学习了：

1. 为什么密码不能明文保存。
2. salt 和 password hash 的作用。
3. token 是什么，以及为什么请求要携带 token。
4. `Authorization: Bearer <token>` 的基本机制。
5. FastAPI `Depends(...)` 如何实现接口鉴权。
6. 用户数据隔离为什么要落实到每个查询和写入。
7. SQLite 唯一约束迁移为什么要检查具体索引字段。
8. 前端为什么需要统一 API client。
9. React Context 如何管理登录状态。
10. 页面级直接 `fetch` 在受保护接口下为什么容易遗漏 token。

## 6. 已记录问题

v0.5 学习和审查中发现的问题已记录到：

```text
docs/code-review-notes.md
```

新增重点问题包括：

1. 前端 API client 没有统一错误处理。
2. `MineScreen` 的 health 检查仍使用独立 fetch。
3. 旧 `default_user` 数据与真实用户 ID 格式不一致。
4. 受保护接口改造后，遗漏页面级 fetch 容易导致 401。
5. 数据库结构迁移判断需要检查约束字段而不是只检查约束存在。

完成 v0.5 后，AI Stock App 已具备：
