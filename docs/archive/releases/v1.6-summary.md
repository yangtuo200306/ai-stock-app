## 3. 已实现功能

### 导航栏统一
- 所有 Tab 和 Stack 统一 `headerShown: false`，消除双层标题
- 子页面（TaskStatus、RecordDetail、ReportDetail）开启原生导航栏，带返回按钮
- 去掉子页面的 SafeAreaView，避免双层间距

### 底部栏美化
- 添加图标（★ ? ☰ ◎）
- 主题色选中效果
- 安全区域适配（`insets.bottom`），底部留白给系统手势
- 关闭 Android 水波纹光效

### 布局修复
- 所有页面统一使用 `react-native-safe-area-context` 的 SafeAreaView（Android 生效）
- WatchlistScreen 容器 `padding` 改为 `paddingHorizontal`，消除顶部多余间距
- 输入框布局优化：匹配列表改为绝对定位，Tag 改为内联与输入框同行

### 交互优化
- 示例按钮改为只填入输入框，不自动发送
- 自选卡片按钮去掉图标，纯文字更简洁
- 应用名改为中文"AI 股票分析"

### 后端修复
- 新增 `_clean_stock_name()` 自动去除 XD（除息）、XR（除权）、DR、ST 等前缀
- `.env` 配置复制到 `backend/` 目录，AI 问股正常调用大模型

## 4. 验证结果

- `npx tsc --noEmit` 编译零错误
- 后端启动无报错
- 核心链路验证通过（注册→添加自选→问股→查看记录→删除）

## 5. 本次修复的问题

| 问题 | 原因 | 修复方式 |
|------|------|---------|
| 双层标题 | Tab/Stack 原生导航栏 + 页面内部标题同时显示 | 统一 `headerShown: false` |
| 内容贴到状态栏 | 关掉原生导航栏后 Android SafeAreaView 不生效 | 改用 `react-native-safe-area-context` |
| 标题栏高度不一致 | WatchlistScreen 容器用了 `padding: 16px` | 改为 `paddingHorizontal` |
| 底部栏太简陋 | 只有文字没有图标 | 添加图标和样式美化 |
| 底部栏图标文字溢出 | 固定高度不够 | 用 `insets.bottom` 动态计算高度 |
| 切换 Tab 有光效 | Android 水波纹 | 用 `PlatformPressable` 关掉 |
| 子页面没有返回按钮 | 所有页面都 `headerShown: false` | 子页面开启原生导航栏 |
| 输入框被键盘挡住 | KeyboardAvoidingView 对 Android 无效 | 改为 `behavior="padding"` |
| 输入框变窄 | 匹配列表和 Tag 占用布局空间 | 匹配列表绝对定位；Tag 内联 |
| 示例按钮直接发送 | onQuickQuestion 调了 handleAsk | 改为只填入输入框 |
| AI 问股不生效 | .env 文件路径不对 | 复制到 backend/ 目录 |
| "AI问AI"按钮 | 图标 AI + 标签 问 AI 重复 | 去掉所有图标 |
| "AI Stock"显示不全 | 英文名被截断 | 改为中文 |
| "XD贵州茅台" | 数据源返回除息标记 | 后端加 `_clean_stock_name()` |
