# v0.7 功能定型与交互补齐总结

## 1. 版本定位

v0.7 的目标是在 UI/UX 大优化前完成产品功能定型，补齐 v0.6 后暴露的操作断点。

本版本重点是功能定型与交互补齐，让 `自选 | 问股 | 记录 | 我的` 四个 Tab 职责稳定，为 v0.8 UI/UX 集中优化做准备。

## 2. 已完成内容

### 2.1 records 增加 updated_at

1. `records` 表新增 `updated_at` TEXT 字段。
2. 旧记录回填 `updated_at = created_at`，保证排序正常。
3. 新增记录时 `created_at` 和 `updated_at` 同步设置。
4. 追问更新同一条 record 时同步刷新 `updated_at = CURRENT_TIMESTAMP`。
5. `GET /api/records` 改为按 `updated_at DESC, id DESC` 排序。
6. `GET /api/records` 和 `GET /api/records/{record_id}` 返回 `updated_at`。
7. 前端记录列表优先展示 `updated_at`，没有时兼容 `created_at`。

### 2.2 问股页增加"新建会话"按钮

1. `AskScreen` 新增"新建会话"按钮。
2. 点击后清空 `sessionId`、`messages`、`latestResult`、`question`、`error`。
3. 不调用后端接口，旧会话保留在记录列表中。
4. 有会话或消息时显示按钮，避免干扰新手。

### 2.3 记录详情支持继续追问

1. `RecordDetailScreen` 在 `record_type === 'ask'` 且存在 `session_id` 时显示"继续追问"按钮。
2. 点击后跳转到问股 Tab，携带 `sessionId` 和 `initialMessages`。
3. 问股页收到路由参数后恢复会话上下文。
4. 不新增后端接口。
5. 不做完整历史会话列表恢复。

### 2.4 未登录态统一引导登录

1. 6 个业务页面（自选、问股、记录列表、记录详情、任务状态、报告详情）接入 `useAuth()`。
2. 未登录时显示统一提示：标题"请先登录"、描述"登录后可以使用自选、问股和记录功能。"、"去登录"按钮。
3. 点击"去登录"跳转到"我的"Tab 的 Login 页面。
4. 我的页和登录页不拦截。

### 2.5 自选页展示最近一次分析摘要

1. `GET /api/stocks` 返回每只自选股最近一条相关 record 摘要。
2. 返回字段包括：`latest_record_id`、`latest_record_type`、`latest_summary`、`latest_updated_at`。
3. 前端 `Stock` 类型增加这些可选字段。
4. 自选卡片展示最近分析摘要（最多两行）和最近更新时间。
5. 没有记录时显示"暂无分析记录"。

### 2.6 报告固定为记录详情的一种类型

1. 自选分析生成的记录类型从 `analysis` 改为 `report`。
2. 记录列表增加 `report` 类型标签，使用独立颜色（`#7c3aed`）。
3. 记录详情支持 `report` 类型展示，保留"查看完整报告"按钮。
4. 旧 `analysis` 记录继续兼容显示。

### 2.7 初步统一 API 错误结构

1. 后端约定错误结构为 `{ error_code, message }`。
2. 覆盖接口：未登录/token 无效、问股输入错误、会话不存在、行情失败、登录注册失败、record/report/task 不存在。
3. 不存在资源统一返回 `HTTPException(status_code=404, ...)`。
4. 前端 `apiGet` / `apiPost` / `apiDelete` 统一处理非 2xx 响应。
5. 错误解析优先级：`detail.message` > `message` > `detail` > 默认文案。
6. 页面 catch 中统一展示 `error.message`。

### 2.8 清理调试字段、重复页面和重复逻辑

1. 删除未挂载的 `SettingsScreen.tsx` 和 `MarketScreen.tsx`。
2. `MineScreen.tsx` 改用 `STORAGE_KEYS.BACKEND_URL`，消除重复 key 定义。
3. `WatchlistScreen.tsx` 合并重复 `useNavigation` 导入。
4. 修复 `AskScreen` 加入自选失败仍提示成功的问题。

### 2.9 抽出 ask service

1. 新增 `backend/app/services/ask_service.py`。
2. 从 `ask.py` 移出 8 个辅助函数：
   - `_build_rule_answer`
   - `_create_ask_session`
   - `_get_ask_session`
   - `_update_ask_session`
   - `_write_ask_message`
   - `_get_ask_messages`
   - `_write_or_update_ask_record`
3. `ask.py` 保留 `APIRouter`、请求模型、响应模型和接口主流程。
4. 不改变接口路径和响应字段。

### 2.10 确认底部导航和页面职责

1. 保持底部 Tab：自选 | 问股 | 记录 | 我的。
2. 不新增底部 Tab。
3. 不恢复大盘页。
4. 各模块职责稳定。

## 3. 审查和修正结果

v0.7 首次审查发现 7 个偏差，均已修复并复审通过：

1. 未登录态缺少"去登录"按钮和跳转逻辑 → 6 个业务页面全部补齐。
2. `ReportDetailScreen` 鉴权顺序不符合计划 → 调整 useEffect 守卫和渲染顺序。
3. `WatchlistScreen` 仍存在重复 `useNavigation` 导入 → 清理别名。
4. 登录失败错误结构未统一 → 改为 `{error_code, message}`。
5. record/report/task 不存在时返回 200 → 改为 `HTTPException(status_code=404)`。
6. 多个页面 catch 使用固定文案 → 改用 `err.message`。
7. `analysis` 和 `report` 使用相同颜色 → `analysis` 改为 `#64748b`。

复审结论：

```text
实施与计划完全相符
```

VS Code diagnostics：无报错。
TypeScript 编译：0 错误。
后端 import 检查：通过。

## 6. 学习重点

本版本重点学习了：

1. 数据库迁移：`ALTER TABLE ADD COLUMN` + 回填旧数据。
2. 排序策略：`ORDER BY updated_at DESC, id DESC` 的双重保障。
3. 服务层拆分：API 层负责主流程，service 层负责业务细节。
4. 未登录态声明式引导：页面自己声明需要登录，而不是等接口报错。
5. 跨 Tab 导航：记录详情跳转问股页并携带参数。
6. 路由参数设计：可选参数支持多个入口进入同一页面。
7. API 错误结构统一：集中管理错误解析，页面不再各自猜格式。
8. 降级机制：AI 失败时规则回答兜底。
9. 兜底回答和字段语义：`answer_type` 和 `ai_status` 的分工。
10. 会话隔离：同一会话固定绑定一只股票。
11. 报告作为记录类型：新数据用新格式，旧数据兼容。
12. 后端聚合查询：自选列表 + 最近摘要由后端一次返回。
13. 死代码清理：删除未挂载页面，确认无残留引用。
14. 前后端消息同步：`session_id` 是前后端约定的对话房间号。

## 7. 已记录问题

v0.7 学习、体验和审查中发现的问题已记录到：

```text
docs/code-review-notes.md
```

新增重点问题包括：

1. 用户 ID 字段类型不完全统一（INTEGER vs TEXT）。
2. `records.updated_at` 依赖 TEXT 时间字符串排序。
3. `GET /api/records` 暂无分页。
4. 记录详情不存在时仍返回 200 风格对象。
5. 记录详情一次性返回全部 messages。
6. ask service 使用下划线函数名但跨文件导入。
7. ask 流程仍非严格单事务。
8. AI 失败未记录后端日志。
9. `answer_type` 和 `ai_status` 语义仍有重叠。
10. 问股页仍无完整会话恢复能力。
11. 未登录引导 UI 重复。
12. token 运行中失效体验仍可优化。
13. 旧 `analysis` 记录需长期兼容。
14. `record_type` 没有枚举约束。
