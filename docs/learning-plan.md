# AI Stock App 学习计划

## 1. 学习目标

本项目的主要目标不是一次性做出完整产品，而是通过参考 `stock-analysis-reference` 原项目，逐步学习并实践一个手机股票分析 APP 的完整开发流程。

核心目标：

- 学会从原项目中提取架构思路，而不是直接照搬代码。
- 学会设计自己的项目结构。
- 学会后端 API、数据库、任务队列、移动端页面、接口联调。
- 学会用 Git 管理自己的项目。
- 学会和 AI 助手协作：先研究、再计划、再执行、再审查。
- 最终让 APP 从最小闭环开始，逐步演进成更完整的股票分析工具。

## 2. 项目边界

当前工作区结构：

```text
d:/ai-stock-analysis/
  stock-analysis-reference/   # 原项目，只读参考
  ai-stock-app/               # 自己开发的新项目
    backend/                  # 后端服务
    mobile-app/               # 手机 APP
    docs/                     # 学习与项目文档
```

边界原则：

- `stock-analysis-reference` 只作为教材阅读，不修改、不提交、不作为自己的项目维护。
- `ai-stock-app` 是自己的项目，后续只在这里初始化 Git 和开发功能。
- 需要参考原项目时，优先学习思想和结构，再在自己的项目中重新实现。
- 不一开始追求原项目全部功能，先完成最小闭环。

## 3. 第一版 APP 目标

第一版目标是完成一个最小可用闭环：

```text
添加股票
  ↓
点击分析
  ↓
后端创建任务
  ↓
APP 查询任务状态
  ↓
生成并保存报告
  ↓
APP 查看报告详情
  ↓
APP 查看历史报告
```

第一版不追求真实行情和复杂 AI 分析，可以先使用 mock 数据生成报告。

## 4. 第一版页面

第一版计划 5 个页面：

| 页面 | 功能 |
|---|---|
| 首页 / 自选股页 | 添加股票、删除股票、展示自选股、触发分析 |
| 分析任务页 | 展示任务状态、进度、失败原因、完成后跳转报告 |
| 报告详情页 | 展示评分、建议、趋势、摘要、风险、技术指标 |
| 历史报告页 | 展示历史报告列表，点击进入详情 |
| 设置页 | 配置后端地址，测试连接 |

## 5. 第一版后端接口

| 接口 | 作用 |
|---|---|
| `GET /api/health` | 健康检查 |
| `GET /api/stocks` | 获取自选股 |
| `POST /api/stocks` | 添加自选股 |
| `DELETE /api/stocks/{code}` | 删除自选股 |
| `POST /api/analysis` | 创建分析任务 |
| `GET /api/analysis/{task_id}` | 查询任务状态 |
| `GET /api/reports` | 获取历史报告列表 |
| `GET /api/reports/{id}` | 获取报告详情 |

## 6. 第一版数据库

第一版使用 SQLite，建议三张表：

### stocks

保存自选股。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| code | 股票代码 |
| name | 股票名称 |
| created_at | 添加时间 |

### analysis_tasks

保存分析任务状态。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| task_id | 任务 ID |
| stock_code | 股票代码 |
| status | pending / processing / completed / failed |
| progress | 进度 |
| message | 当前消息 |
| report_id | 报告 ID |
| error | 错误信息 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

### reports

保存分析报告。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| stock_code | 股票代码 |
| stock_name | 股票名称 |
| price | 当前价格 |
| score | 评分 |
| action | 操作建议 |
| trend | 趋势判断 |
| summary | 分析摘要 |
| risks_json | 风险 JSON |
| indicators_json | 技术指标 JSON |
| created_at | 创建时间 |

- 保存报告。

9. 推送提醒。

## 9. 和 AI 助手协作方式

为了更好学习，建议每次只做一个小目标。

常用表达：

| 场景 | 推荐说法 |
|---|---|
| 想了解代码 | 进入研究模式，帮我解释这个文件 |
| 想比较方案 | 进入创新模式，比较这几个方案 |
| 想开始设计 | 进入计划模式，帮我制定详细计划 |
| 想写代码 | 进入执行模式，按刚才计划实现 |
| 想检查结果 | 进入审查模式，检查是否符合计划 |
| 不知道下一步 | 当前做到这里了，下一步建议做什么 |

协作原则：

- 不急着写代码，先搞清楚目标。
- 涉及代码变更前先计划。
- 每次只做一个可验证的小任务。
- 写完代码要运行或检查。
- 文档只有明确允许时才生成或更新。
